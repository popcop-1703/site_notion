Практическая работа №2 Active Directory.

Занятие 2 - Active Directory

Задание:

1) Установить роли и компоненты на оба Windows Server

2) Повысить роль первого WS до роли контроллера домена

3) Сконфигурировать второй WS как второстепенный (резервный) контроллер домена

4) Ввести Win10 в домен

5) Применить скрипт создания пользователей

Методический материал:

Часть 1. Установка ролей и компонент

Роли и компоненты -- дополнительные программы, обеспечивающие фунциональность сервера. Для реализации контроллера домена понадобится некоторые роли.

==1.6 При выборе ADDS нам предлагают установить дополнительные компоненты. Соглашаемся (Add Featured)==

==1.7 Выбираем пункт DHCP==

==1.8 Устанавливаем компоненты==

==1.9 Выбираем пункт DNS==

==1.10 Устанавливаем компоненты==

==1.11 Выбрав нужые роли, идём далее. NEXT==

==1.12 Нужные компоненты мы уже выбрали. NEXT==

==1.13 Далее идет описание устанавливаемых ролей. NEXT==

==1.14 NEXT==

==1.15 NEXT==

В данном меню можно сразу нажать "promote this servet to a domain controller", чтобы открыть окно повышения роли.

Часть 5. Применение скриптов

Скачайте скрипт .https://disk.yandex.ru/d/sC2jjnK46ey6rA Это PoSH скрипт, который необходимо применить на первом контроллере домена (dc1).

==5.1 Откройте PowerShell ISE==

==5.2 Откройте скачанный файл==

==5.3 Выполните скрипт (F5)==

==5.4 В нижнем меню ошибок появляться не должно. В случае возникновения текста проверяйте, на что именно ругается ISE==

==5.5 Проверьте наполнение NTDS с помощью ADUC. Должны появиться новые OU, группы и пользователи==

Не забывайте, что оснастки не обновляются самостоятельно. Чтобы увидеть изменения в случае, если при применении скрипта ADUC уже был запущен, нажмите ПКМ на pt.local и нажмите "refresh", чтобы обновить представление.
