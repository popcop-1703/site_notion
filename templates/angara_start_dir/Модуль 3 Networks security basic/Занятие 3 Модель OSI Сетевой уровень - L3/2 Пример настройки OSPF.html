2. Пример настройки OSPF

Топология сети

Характеристики оборудования для самостоятельного построения стенда:

  • Cisco IOL: Router - L3 образ. 512mb RAM;
  • Cisco IOL: Switch - L2 образ. 256mb RAM;
  • Mikrotik - Mikrotik ver.6.39;
  • Linux: Ubuntu 16.04 Server. 1024mb RAM, Ethernet 2;
  • VPC.

Видео-инструкция по настройке OSPF.

Настройка OSPF

Роутер OSPF34

Router>en
Router#conf t
Router(config)#router ospf 1
Router(config-router)#network 172.16.0.0 0.0.255.255 area 0
  • router ospf id - запуск и переход в режим конфигурации OSPF. Последнее число — это PID процесса, может отличаться на разных роутерах, но исключительно для удобства лучше задавать одно и то же число.
  • network ip wildcard-mask area area-id - анонсирование непосредственно присоединенной сети к маршрутизатору. wildcard mask — маска, которая указывает с помощью 0 какая часть из указанной сети должна совпадать, а с помощью 1 какая часть сети может быть произвольной. area-id — идентификатор зоны, в которой будет работать интерфейс маршрутизатора.
Примечание! В отличие от протокола RIP с помощью wildcard маски можно указать, какая часть сети может быть произвольной. Одной командой network 172.16.0.0 0.0.255.255 можно сразу добавить сети 172.16.3.0, 172.16.9.0, 172.16.10.0, 172.16.17.0 в анонс.

Аналогично для остальных роутеров в зоне RIP.

Для вывода маршрутов, которые пришли с помощью протокола OSPF воспользуемся командой show ip route ospf.

Соединение зон

Сейчас все зоны (Static routes, RIP, OSPF) функционируют отдельно друг от друга, то есть нельзя выполнить команду ping из одной зоны в другую. Для решения нам понадобится:

  • Пробросить статические маршруты в OSPF и RIP.
  • Пробросить маршруты RIP в OSPF.
  • Пробросить маршруты OSPF в RIP.
  • Добавить default route на все роутеры в зоне Static.

В связи с этим на роутере Main_Router одновременно будет работать OSPF и RIP.

Для начала следует настроить ip адреса на Main_Router, а также включить RIP и OSPF.

Main_Router>en
Main_Router#conf t
Main_Router(config)#int e1/1
Main_Router(config-if)#no shutdown
Main_Router(config-if)#ip addr 192.168.17.2 255.255.255.252
Main_Router(config-if)#int e1/2
Main_Router(config-if)#no shutdown
Main_Router(config-if)#ip addr 172.16.17.2 255.255.255.252
Main_Router(config-if)#exit
Main_Router(config)#router rip
Main_Router(config-router)#version 2
Main_Router(config-router)#network 192.168.17.0
Main_Router(config-router)#router ospf 1
Main_Router(config-router)#network 172.16.17.0 0.0.0.3 area 0

1. Проброс статических маршрутов в OSPF и RIP.

Main_Router(config)#router rip
Main_Router(config-router)#redistribute static
Main_Router(config-router)#router ospf 1
Main_Router(config-router)#redistribute static subnets

2. Проброс маршрутов RIP в OSPF.

Main_Router(config)#router rip
Main_Router(config-router)#redistribute ospf 1 metric 1

3. Проброс маршрутов OSPF в RIP.

Main_Router(config)#router ospf 1
Main_Router(config-router)#redistribute rip subnets

4. Добавление default route на все роутеры в зоне Static

Default route - адрес, на который пакет отправляется в том случае, если маршрут к сети назначения пакета не известен. Шлюз по умолчанию задаётся записью в таблице маршрутизации вида «сеть 0.0.0.0 с маской сети 0.0.0.0»

В нашем случае шлюзом будет выступать роутер Main_Router, поэтому следует прописать 8-ми нулевый маршрут до него.

Рассмотрим ветку из роутеров R4-2, R4-1, Main_Router.

R4-2

R4-2(config)#ip route 0.0.0.0 0.0.0.0 10.4.2.2

R4-1

R4-1(config)#ip route 0.0.0.0 0.0.0.0 10.4.1.2

Если роутер не найдет в своей таблицы маршрутизации маршрут к сети назначения, то он перенеправит его на роутер R4-1, который в свою очередь перенаправит его на роутер Main_Router. Отсюда пакеты могу пойти в зону RIP, OSPF, VLAN или Интернет.

Проверить правильность работы можно при помощи трассировки.
