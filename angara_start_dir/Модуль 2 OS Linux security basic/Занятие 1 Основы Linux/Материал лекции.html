Файловая система Linux

Представляет собой древовидную структуру, т.к. все файлы расположены в виде дерева и его ветвей. Самый верхний каталог называется корневым.

Стандартные каталоги

  • / — главный каталог, содержащий в себе всю иерархию системы
  • /boot — директория загрузки системы
  • /root и /home — домашние директории, использующиеся пользователями
  • /bin — директория хранения исполняемых файлов
  • /etc — директория хранения конфигураций общей системы
  • /dev — директория хранения физических устройств, связанных с компьютером
  • /mnt — каталог для временного подключения файловых систем
  • /var — директория файлов переменных данных
  • /tmp — каталог предназначенный для временных файлов
  • /opt — директория дополнительных приложений

Работа с командной строкой

CLI (Command Line Interface) — текстовый интерфейс для работы с операционной системой

Sh — командная оболочка по умолчанию

Bash — более приятная и удобная для пользователя оболочка с расширенным функционалом

Адресация

  • Абсолютная — путь к каталогу указывается, начиная с корневого и далее вниз по дереву
  • Относительная — путь к каталогу указывается, относительно текущего каталога

Использование команд

Существует команда и её параметры.
Параметры делятся на
опции (как должна быть выполнена операция) и аргументы (относительно чего выполняется команда).
Изменяя опции и аргументы можно изменять поведение программы.

Основные команды

  1. cd (change directory) — используется для перехода в директорию
    • cd .. — переход на уровень выше
    • cd / — переход в корневой каталог
    • cd path/to/dir — переход в указанный каталог
    • cd - — переход в предыдущий каталог
  1. Просмотр содержимого в файлах
    • cat (concatenate) — вывод содержимого файла в командной строке
    • tac — вывод файла начиная с последней строки
  1. pwd — вывод абсолютного пути к текущему каталогу
  1. touch — создание нового пустого файла
  1. rm file — удалить файл
    • rm -r folder — удалить каталог
    • rm -rf folder — удалить каталог и его содержимого
    • rm -v file — вывод информации об удаляемых файлах
  1. history — история введённых команд
  1. ls — вывод содержимого текущего каталога
    • ls -l — вывод содержимого каталога длинным списком
    • ls -a — вывод содержимого вместе со скрытыми файлами и каталогами
    • ls -t — вывод списка файлов по дате создания, начиная с самого нового
    • ls -r — вывод содержимого каталога в обратном порядке
    • ls -h — вывод размера файлов (используя вместе с опцией -l)
  1. mkdir newfolder — создать новый каталог c указанием название нового каталога
  1. cp что куда — скопировать файл/директорию в указанный файл/директорию
  1. mv [опции] что куда — перемещение файлов/директорий в другую директорию

Фильтры вывода

  • head — вывод первых строк файла

    head -20 file1
  • tail — вывод последних строк файла

    tail -20 file1
  • less — вывод файла по одной странице за раз
  • more — устаревший аналог less, может читать только вперед
  • nl — вывод с добавлением номера строки
  • sort — вывод строк файла в определенном порядке (по умолчанию — по алфавиту)

    Опции sort:
    • -r — сортировка в обратном по рядке
    • -n — сравнение в соответствии с числовым значением
    • -u — вывод только уникальных строк
    • -b — игнорировать пробелы
    • -d — учитывать только пробелы и буквенно-цифровые символы
    • -o — записать результат в файл, вместо вывода в консоль
  • uniq — вывод строк без повторения

    Опции uniq:
    • -u — вывод только строк без повтора
    • -d — вывод только повторяющихся строк

Пакетный менеджер

Набор программных средств, которые автоматизируют процесс установки, обновления, настройки и удаления компьютерных программ

apt

Консольная утилита, выполняющая роль поисковика и загрузчика пакетов из репозитория

  • apt-get install program — установить/обновить программу
  • apt-get remove program — удалить программу, сохранив файлы конфигурации
  • apt-get purge program — удалить программу и ее файлы конфигурации
  • apt-get check — проверить зависимости
  • apt-get autoclean — удалить старые архивные пакеты

dpkg

Низкоуровневый пакетный менеджер для работы с локальными DEB пакетами
Не может самостоятельно разрешать зависимости и скачивать пакеты из репозиториев

  • dpkg -S /bin/netstat — поиск пакета и к какому дистрибутиву он относится
  • dpkg -L net-tools — просмотр состава пакета или дистрибутива
  • dpkg --purge foo — удалить пакет и его файлы конфигурации
  • dpkg --remove foo — удалить пакет , сохранив файлы конфигурации

yum

Высокоуровневый пакетный менеджер, написанный для систем RED HAT

  • yum install program — установка указанной утилиты
  • yum update program — обновление указанной утилиты
  • yum list all — показать установленные утилиты
  • yum remove program — удаление указанной утилиты

Права доступа

В системе Linux есть несколько видлв пользователей:

  1. Обычные пользователи

    Они создаются системным администратором. Каждый пользователь имеет числовой идентификатор пользователя, называемый UID
  1. Системные пользователи

    Системные пользователи создаются самой операционной системой. Скорее всего они используются какими-либо приложениями
  1. Root

    Данный пользователь имеет UID = 0
    Root может выполнять все операции на низком уровне, запись в критическую область памяти и прочее
    Работать с
     root правами не рекомендуется

Чтобы изменить права на файл в linux вы можете использовать утилиту chmod.

$ chmod опции категория действие флаг файл

  • Чтение (r) - разрешает получать содержимое файла, но на запись нет. Для каталога позволяет получить список файлов и каталогов, расположенных в нем;
  • Запись (w)- разрешает записывать новые данные в файл или изменять существующие, а также позволяет создавать и изменять файлы и каталоги;
  • Выполнение (x)- вы не можете выполнить программу, если у нее нет флага выполнения. Этот атрибут устанавливается для всех программ и скриптов, именно с помощью него система может понять, что этот файл нужно запускать как программу.

  • u - владелец файла;
  • g - группа файла;
  • o - другие пользователи.

Например, дать всем пользователям полный доступ к файлу test5

chmod ugo+rwx test5

  • uniq - с помощью этой команды можно вывести строки без повторения
