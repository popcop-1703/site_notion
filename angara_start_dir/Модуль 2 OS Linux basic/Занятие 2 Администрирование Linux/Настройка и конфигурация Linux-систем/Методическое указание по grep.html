Основные возможности grep включают:

  1. Поиск по строкам текста: grep позволяет искать строки текста в файлах или стандартном вводе, которые соответствуют определенному шаблону.
  1. Использование регулярных выражений: grep поддерживает использование регулярных выражений для более гибкого поиска текста. Регулярные выражения позволяют создавать шаблоны, которые описывают последовательности символов, которые вы хотите найти.
  1. Поиск в рекурсивных каталогах: С помощью опции r (или -recursive) grep может рекурсивно искать во всех подкаталогах указанного каталога.
  1. Вывод номеров строк: Опция n (или -line-number) позволяет grep выводить номера строк, содержащих совпадения.
  1. Игнорирование регистра: Опция i (или -ignore-case) делает поиск регистронезависимым, что означает, что grep будет игнорировать регистр символов при поиске совпадений.
  1. Вывод только совпадающих строк: Опция o (или -only-matching) позволяет grep выводить только сами совпадающие строки, а не весь текст строки.
  1. Вывод контекста: Опции A (или -after-context) и B (или -before-context) позволяют grep выводить строки контекста после и до совпадения соответственно.
  1. Вывод не совпадающих строк: Опция v (или -invert-match) позволяет grep выводить строки, которые не содержат совпадений.
  1. Поиск в определенных типах файлов: С помощью опции r grep может ограничивать поиск только определенными типами файлов, например, только текстовыми файлами.
  1. Поддержка Unicode: grep может работать с символами Unicode, что позволяет осуществлять поиск в текстах на различных языках и с использованием различных символов.
  1. Поиск в нескольких файлах: grep позволяет одновременно искать в нескольких файлах, передавая их имена в качестве аргументов.

1. Grep — мощное поисковое приложение, поставляется на каждом дистрибутиве Linux.

grep --version | grep grep

grep (GNU grep) 2.5.1-FreeBSD

Команда grep --version используется для вывода информации о версии программы grep, которая является утилитой для поиска текста в файлах. Результат этой команды включает в себя информацию о версии grep.

Команда grep grep применяется к выводу grep --version и используется для фильтрации результатов, оставляя только строки, содержащие слово "grep".

Таким образом, после выполнения этой команды вы получите информацию о версии программы grep, и только те строки, в которых упоминается само слово "grep". Это может быть полезно, если вам нужно быстро проверить версию grep в выводе команды, при этом оставив другую информацию.

Это важно в связи с

man grep | grep -iB 2 freebsd
Команда grep -iB 2 freebsd применяется к выводу руководства man grep и используется для фильтрации результатов:
  • grep: Утилита для поиска текста в выводе команд.
  • i: Опция i делает поиск регистронезависимым, то есть игнорирует регистр символов.
  • B 2: Опция B указывает, что перед каждым совпадением выводится указанное количество строк перед строкой совпадения. В данном случае 2 означает, что перед каждым совпадением будет выведено 2 строки.
  • freebsd: Это строка, которую grep ищет в выводе руководства man grep.

Таким образом, после выполнения этой команды вы увидите в руководстве по grep только те строки, которые содержат слово "freebsd", а также две строки перед каждым совпадением, чтобы дополнительно контекстуализировать информацию. Это может быть полезно, если вам нужна информация о том, как grep работает в контексте операционной системы FreeBSD.

2. Для начала о том как мы обычно grep’аем файлы.

Используя cat:

cat /var/run/dmesg.boot | grep CPU:

CPU: Intel® Core™2 Quad CPU Q9550 @ 2.83GHz (2833.07-MHz K8-class CPU)

Команда cat /var/run/dmesg.boot используется для вывода содержимого файла /var/run/dmesg.boot, который обычно содержит сообщения, отправляемые ядром операционной системы при загрузке.

Команда grep CPU: применяется к выводу содержимого файла dmesg.boot и используется для фильтрации результатов:

  • grep: Это утилита для поиска текста в выводе команд.
  • CPU:: Это строка, которую grep ищет в содержимом файла dmesg.boot.

Таким образом, после выполнения этой команды вы увидите только те строки из содержимого файла dmesg.boot, которые содержат фразу "CPU:". В контексте файла dmesg.boot это могут быть сообщения, касающиеся оборудования процессора, его характеристик или конфигурации. Это может быть полезно, если вы хотите получить информацию о процессоре или его работе после загрузки операционной системы.

Но зачем? Ведь можно и так:

grep CPU: /var/run/dmesg.boot

CPU: Intel® Core™2 Quad CPU Q9550 @ 2.83GHz (2833.07-MHz K8-class CPU)

Команда grep CPU: /var/run/dmesg.boot используется для поиска всех строк в файле /var/run/dmesg.boot, содержащих фразу "CPU:".
  • grep: Это утилита для поиска текста в файле.
  • CPU:: Это строка, которую grep ищет в файле /var/run/dmesg.boot.

После выполнения этой команды выводом будут все строки из файла dmesg.boot, в которых содержится фраза "CPU:". В контексте файла dmesg.boot это могут быть сообщения, связанные с оборудованием процессора, его характеристиками или конфигурацией.

Или вот так (ненавижу такую конструкцию):

</var/run/dmesg.boot grep CPU:

CPU: Intel® Core™2 Quad CPU Q9550 @ 2.83GHz (2833.07-MHz K8-class CPU)

Команда grep 'CPU:' </var/run/dmesg.boot используется для поиска всех строк в файле /var/run/dmesg.boot, содержащих фразу "CPU:".
  • grep: Это утилита для поиска текста в файле или стандартном вводе.
  • 'CPU:': Это строка, которую grep ищет в файле /var/run/dmesg.boot.
  • </var/run/dmesg.boot: Это оператор перенаправления, который отправляет содержимое файла /var/run/dmesg.boot в стандартный ввод команды grep.

После выполнения этой команды все строки из файла dmesg.boot, содержащие фразу "CPU:", будут выведены в стандартный вывод терминала. В контексте файла dmesg.boot это могут быть сообщения, связанные с оборудованием процессора, его характеристиками или конфигурацией, записанные в этот файл при загрузке операционной системы.

Зачем-то считаем отобранные строки с помощью wc:

grep WARNING /var/run/dmesg.boot | wc -l
Команда grep WARNING /var/run/dmesg.boot используется для поиска всех строк в файле /var/run/dmesg.boot, содержащих фразу "WARNING".
  • grep: Это утилита для поиска текста в файле или стандартном вводе.
  • WARNING: Это строка, которую grep ищет в файле /var/run/dmesg.boot.
  • /var/run/dmesg.boot: Это путь к файлу, в котором вы хотите произвести поиск.

После выполнения этой команды все строки из файла dmesg.boot, содержащие фразу "WARNING", будут выведены в стандартный вывод терминала.

Команда wc -l используется для подсчета количества строк, переданных в стандартный ввод.

  • wc: Это утилита для подсчета количества слов, строк и символов в текстовом файле или стандартном вводе.
  • l: Опция l указывает wc подсчитать только количество строк во вводе.

Таким образом, после выполнения этой команды будет подсчитано количество строк, содержащих фразу "WARNING" в файле dmesg.boot, и результат будет выведен в стандартный вывод.

Хотя можно:

grep WARNING /var/run/dmesg.boot -c
Команда grep WARNING /var/run/dmesg.boot -c используется для подсчета количества строк в файле /var/run/dmesg.boot, содержащих фразу "WARNING".
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • WARNING: Это строка, которую grep ищет в файле /var/run/dmesg.boot.
  • /var/run/dmesg.boot: Путь к файлу, в котором осуществляется поиск.
  • c: Опция c указывает grep на вывод только количества строк, содержащих совпадение.

После выполнения этой команды grep найдет все строки в файле dmesg.boot, содержащие фразу "WARNING", и выведет только количество этих строк, а не сами строки.

3. И приступим к поискам:

Опция -w позволяет искать по слову целиком:

grep -w 'seven' test.txt
Команда grep -w 'seven' test.txt используется для поиска вхождений строки "seven" в файле test.txt, при этом учитывая только целые слова.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • w: Опция w указывает grepу искать только целые слова. Это означает, что он будет считать совпадением только те строки, в которых слово "seven" отделено от других символов пробелами, табуляциями или началом или концом строки.
  • 'seven': Это строка, которую grep ищет в файле test.txt.
  • test.txt: Имя файла, в котором производится поиск.

Таким образом, после выполнения этой команды grep просматривает содержимое файла test.txt и выводит все строки, в которых встречается слово "seven" как отдельное слово. Это означает, что строки, содержащие слова вроде "seventeen" или "seventy", не будут учтены в результатах, так как они содержат "seven" как часть слова, а не отдельное слово.

А если нужно по началу или концу слова?

grep '\<seven' test.txt
Команда grep '\ используется для поиска всех строк в файле test.txt, которые начинаются с подстроки "seven" как целого слова.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • \<seven: Это регулярное выражение, которое указывает grepу искать слова, начинающиеся с подстроки "seven".

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, в которых слово "seven" является отдельным словом и находится в начале строки или сразу после пробела. Например, строки, содержащие "sevenfold" или "seventeen", не будут считаться совпадениями, так как они не начинаются с "seven".

grep 'seven\>' test.txt
Команда grep 'seven\>' test.txt используется для поиска всех строк в файле test.txt, которые заканчиваются подстрокой "seven" как целого слова.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • 'seven\>': Это регулярное выражение, которое указывает grepу искать слова, заканчивающиеся подстрокой "seven".
  • test.txt: Имя файла, в котором производится поиск.

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, в которых слово "seven" является отдельным словом и находится в конце строки или перед пробелом. Например, строки, содержащие "seventeen" или "seventh", не будут считаться совпадениями, так как они не заканчиваются на "seven".

Команда grep '^seven' test.txt используется для поиска всех строк в файле test.txt, которые начинаются с подстроки "seven".
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • '^seven': Это регулярное выражение, которое указывает grepу искать строки, начинающиеся с подстроки "seven".

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, которые начинаются с слова "seven". строки, которые содержат "seven" в другом месте, но не в начале, не будут считаться совпадениями.

grep 'seven$' test.txt
Команда grep 'seven$' test.txt используется для поиска всех строк в файле test.txt, которые заканчиваются подстрокой "seven".
  • grep: Утилита для поиска текста в файле или стандартном вводе.

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, в которых слово "seven" находится в конце строки. строки, которые содержат "seven" в другом месте, но не в конце, не будут считаться совпадениями.

Хотите увидеть строки в окрестности искомой?

grep -C 1 twentyseven test.txt
Команда grep -C 1 twentyseven test.txt используется для поиска всех строк в файле test.txt, содержащих фразу "twentyseven", а также вывода одной строки контекста до и после каждого совпадения.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • C 1: Опция C указывает grepу вывести указанное количество строк контекста перед и после каждого совпадения. В данном случае 1 указывает, что нужно вывести одну строку контекста.
  • twentyseven: Это строка, которую grep ищет в файле test.txt.
  • test.txt: Имя файла, в котором производится поиск.

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, содержащие фразу "twentyseven", а также одну строку контекста до и после каждого совпадения. Это позволяет увидеть окружающий контекст в каждом месте, где встречается искомая фраза.

А ещё мы умеем так

grep "twenty[1-4]" test.txt
Команда grep "twenty[1-4]" test.txt используется для поиска всех строк в файле test.txt, содержащих подстроки "twenty1", "twenty2", "twenty3" или "twenty4".
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • "twenty[1-4]": Это регулярное выражение, называемое регулярным выражением с символьными классами. Оно указывает grepу на поиск строки, содержащей "twenty" с последующей цифрой, которая может быть 1, 2, 3 или 4.
  • test.txt: Имя файла, в котором производится поиск.

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, в которых содержатся подстроки "twenty1", "twenty2", "twenty3" или "twenty4". Это может быть полезно, если вам нужно найти строки, содержащие конкретные числа после слова "twenty" в вашем файле.

И наоборот исключая эти

grep "twenty[^1-4]" test.txt
Команда grep "twenty[^1-4]" test.txt используется для поиска всех строк в файле test.txt, содержащих подстроки "twenty" с последующим символом, отличным от цифр 1, 2, 3 или 4.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • "twenty[^1-4]": Это регулярное выражение, которое указывает grep на поиск строки, содержащей "twenty", за которой следует символ, отличный от цифр 1, 2, 3 или 4. Символ ^ внутри квадратных скобок указывает на отрицание, то есть исключает из поиска указанные символы.
  • test.txt: Имя файла, в котором производится поиск.

Таким образом, после выполнения этой команды grep будет выводить строки из файла test.txt, в которых содержатся подстроки "twenty" с последующими символами, отличными от цифр 1, 2, 3 или 4. Это может быть полезно, если вам нужно найти строки, содержащие "twenty" с последующими символами, отличными от указанных.

4. Разумеется grep поддерживает и прочие базовые квантификаторы, метасимволы и другие прелести регулярок

Пару практических примеров:

cat /etc/resolv.conf

Файл /etc/resolv.conf содержит настройки DNS (Domain Name System) для системы. DNS отвечает за преобразование доменных имен (например, www.example.com) в IP-адреса, используемые для связи с удаленными хостами в сети Интернет.

Обычно этот файл содержит информацию о DNS-серверах, которые ваша система использует для разрешения доменных имен. Вот как выглядит типичное содержимое файла /etc/resolv.conf:

Generated by NetworkManager
search example.com
nameserver 192.168.1.1
nameserver 8.8.8.8

Отбираем только строки с ip:

Команда grep -E "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}" /etc/resolv.conf используется для поиска в файле /etc/resolv.conf строк, содержащих IPv4-адреса.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • E: Опция E (или -extended-regexp) указывает grep использовать расширенные регулярные выражения.
  • "[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}\.[0-9]{1,3}": Это регулярное выражение, которое указывает grep на поиск строк, содержащих IPv4-адреса. Регулярное выражение состоит из четырех блоков [0-9]{1,3}, каждый из которых соответствует одному числу от 0 до 255 (это определяется {1,3}, что означает от 1 до 3 цифр). Каждый блок разделен точкой \..
  • /etc/resolv.conf: Это путь к файлу, в котором производится поиск.

После выполнения этой команды grep выведет строки из файла /etc/resolv.conf, содержащие IPv4-адреса. Это может быть полезно, если вы хотите найти IP-адреса DNS-серверов, указанных в файле resolv.conf.

5. Здесь остановимся на инвертировании поиска ключом -v

Допустим нам нужно выполнить «ps -afx | grep ttyv»

ps -afx | grep ttyv
Команда ps -afx | grep ttyv используется для вывода информации о процессах, запущенных в системе, и фильтрации вывода с помощью grep, чтобы найти процессы, связанные с управлением виртуальными терминалами (ttyv).
  • ps: Утилита для вывода информации о процессах.
  • afx: Опции a, f и x используются для вывода всех процессов (даже тех, которые не принадлежат текущему терминалу), а также для вывода информации о дереве процессов и неинтерактивных процессов.
  • |: Оператор конвейера (pipe), который передает вывод команды слева от него как ввод для команды справа от него.
  • grep ttyv: Это команда grep, которая фильтрует вывод ps, оставляя только строки, содержащие подстроку "ttyv".

Когда вы выполняете эту команду, она выводит список процессов, у которых в имени или аргументах командной строки содержится подстрока "ttyv". В результате вы увидите только те процессы, которые связаны с виртуальными терминалами (ttyv). Это могут быть процессы управления входом/выходом для виртуальных консолей на вашей системе.

Всё бы ничего, но строка «48798 2 S+ 0:00.00 grep ttyv» нам не нужна. Используем -v

ps -afx | grep ttyv | grep -v grep
Команда ps -afx | grep ttyv | grep -v grep используется для вывода информации о процессах, связанных с управлением виртуальными терминалами (ttyv), и исключения из вывода строки, содержащие саму команду grep, чтобы избежать ложных совпадений.
  • ps: Утилита для вывода информации о процессах.
  • afx: Опции a, f и x используются для вывода всех процессов (даже тех, которые не принадлежат текущему терминалу), а также для вывода информации о дереве процессов и неинтерактивных процессов.
  • |: Оператор конвейера (pipe), который передает вывод команды слева от него как ввод для команды справа от него.
  • grep ttyv: Это первая команда grep, которая фильтрует вывод ps, оставляя только строки, содержащие подстроку "ttyv".
  • grep -v grep: Это вторая команда grep, которая фильтрует вывод предыдущей команды, исключая строки, содержащие подстроку "grep". Это нужно для того, чтобы избежать ложных совпадений, поскольку команда grep ttyv сама по себе будет отображена в выводе как процесс, содержащий подстроку "ttyv".

Таким образом, после выполнения этой команды вы получите только строки, соответствующие процессам, связанным с виртуальными терминалами (ttyv), и исключите строки, содержащие команду grep, чтобы избежать ложных совпадений.

Некрасивая конструкция? Потрюкачим немного:

ps -afx | grep "[t]tyv"
Команда ps -afx | grep "[t]tyv" используется для вывода информации о процессах, связанных с управлением виртуальными терминалами (ttyv), при этом исключая сам процесс поиска grep из вывода.
  • ps: Утилита для вывода информации о процессах.
  • afx: Опции a, f и x используются для вывода всех процессов (даже тех, которые не принадлежат текущему терминалу), а также для вывода информации о дереве процессов и неинтерактивных процессов.
  • |: Оператор конвейера (pipe), который передает вывод команды слева от него как ввод для команды справа от него.
  • grep "[t]tyv": Это команда grep, которая фильтрует вывод ps, оставляя только строки, содержащие подстроку "ttyv". Однако, в отличие от обычного поиска grep ttyv, здесь используется трюк с регулярным выражением: [t]tyv. Это позволяет избежать вывода самой команды grep в результате поиска, так как регулярное выражение [t]tyv соответствует строке, содержащей символ t, за которым следует tyv. Таким образом, строка, содержащая [t]tyv, фактически не будет содержать точно такую же последовательность символов, как сама команда grep, что позволяет избежать ложных совпадений.

Таким образом, после выполнения этой команды вы получите только строки, соответствующие процессам, связанным с виртуальными терминалами (ttyv), и исключите строки, содержащие саму команду grep, чтобы избежать ложных совпадений.

Отберём строки с заглавными символами:

grep "[[:upper:]]" test.txt
Команда grep "[[:upper:]]" test.txt используется для поиска строк в файле test.txt, содержащих хотя бы одну заглавную букву.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • "[[:upper:]]": Это регулярное выражение, которое соответствует одной заглавной букве. [[:upper:]] представляет собой класс символов в регулярных выражениях, который соответствует любой заглавной букве в ASCII.

После выполнения этой команды grep выведет все строки из файла test.txt, содержащие хотя бы одну заглавную букву. Таким образом, это позволяет быстро найти и выделить строки, содержащие заглавные буквы в файле.

8. Ну и ещё пару трюков для затравки.

Нужно из нашего тестового файла выбрать строки содержащие six или seven или eight:

Пока всё просто:

grep -E "(six|seven|eight)" test.txt
Команда grep -E "(six|seven|eight)" test.txt используется для поиска строк в файле test.txt, содержащих подстроки "six", "seven" или "eight".
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • E: Опция E (или -extended-regexp) указывает grep использовать расширенные регулярные выражения.
  • "(six|seven|eight)": Это регулярное выражение, которое соответствует одной из трех строк: "six", "seven" или "eight". Круглые скобки () используются для группировки альтернативных выражений, а символ | означает "или" в регулярных выражениях.
  • test.txt: Это имя файла, в котором выполняется поиск.

После выполнения этой команды grep выведет все строки из файла test.txt, содержащие хотя бы одну из трех подстрок: "six", "seven" или "eight". Таким образом, это позволяет быстро найти и выделить строки, содержащие одну из указанных подстрок в файле.

А теперь только те строки в которых six или seven или eight встречаются несколько раз. Эта фишка именуется Backreferences

grep -E "(six|seven|eight).*\1" test.txt
Команда grep -E "(six|seven|eight).*\1" test.txt используется для поиска строк в файле test.txt, которые содержат одну из строк "six", "seven" или "eight", за которой следует любая последовательность символов, а затем повторяется та же строка (то есть, повтор).
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • E: Опция E (или -extended-regexp) указывает grep использовать расширенные регулярные выражения.
  • "(six|seven|eight).*\1": Это регулярное выражение ищет строки, которые содержат одну из строк "six", "seven" или "eight", за которой следует любая последовательность символов (.*), а затем повторяется та же строка (\1). Символ \1 соответствует тексту, совпадающему с содержимым первой группы скобок (six|seven|eight) в выражении.
  • test.txt: Это имя файла, в котором выполняется поиск.

После выполнения этой команды grep выведет все строки из файла test.txt, которые содержат подстроку "six", "seven" или "eight", за которой следует любая последовательность символов, а затем повторяется та же строка. Таким образом, это позволяет быстро найти и выделить строки, содержащие повторяющиеся последовательности "six", "seven" или "eight".

Использование POSIX-классов не спасает:

grep "[[:blank:]]504[[:blank:]]" test.txt
Команда grep "[[:blank:]]504[[:blank:]]" test.txt используется для поиска строк в файле test.txt, которые содержат подстроку "504", окруженную пробельными символами (пробелами или табуляциями).
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • "[[:blank:]]504[[:blank:]]": Это регулярное выражение ищет строки, в которых подстрока "504" находится между пробельными символами [[:blank:]]. [[:blank:]] представляет собой класс символов в регулярных выражениях, который соответствует любому пробельному символу, включая пробелы и табуляции.
  • test.txt: Имя файла, в котором выполняется поиск.

После выполнения этой команды grep выведет все строки из файла test.txt, в которых подстрока "504" находится между пробельными символами. Таким образом, это позволяет быстро найти и выделить строки, содержащие конкретное значение "504", которое окружено пробельными символами.

На помощь приходит конструкция [CTRL+V][TAB]:

root@nm3:/ # grep "     504     " test.txt
Команда grep " 504 " test.txt используется для поиска строк в файле test.txt, которые содержат точно строку "504", окруженную шестью пробелами.
  • grep: Утилита для поиска текста в файле или стандартном вводе.
  • " 504 ": Это строка, которую grep ищет в файле test.txt. Заметьте, что между цифрой "504" и пробелами в выражении имеется по шесть пробелов для точного соответствия.
  • test.txt: Имя файла, в котором выполняется поиск.

После выполнения этой команды grep выведет все строки из файла test.txt, в которых точно присутствует строка "504", окруженная шестью пробелами. Это позволяет быстро найти строки с определенным значением "504", когда они представлены в формате, где количество пробелов между значениями фиксировано.

9. Разумеется, grep умеет искать в файлах/каталогах и, разумеется, рекурсивно.

Найдём в исходниках код, где разрешается использование Intel’ом сторонних SFP-шек. Как пишется allow_unsupported_sfp или unsupported_allow_sfp не помню. Ну да и ладно — это проблемы grep’а:

grep -rni allow /usr/src/sys/dev/ | grep unsupp
Эта команда используется для поиска строк, содержащих слово "allow" в файлах, находящихся в директории /usr/src/sys/dev/, а затем фильтрации результатов с помощью grep, чтобы найти только те строки, содержащие подстроку "unsupp".
  • grep -rni allow /usr/src/sys/dev/: Эта часть команды grep ищет рекурсивно (r) в директории /usr/src/sys/dev/ все строки, содержащие слово "allow". Опция n позволяет выводить номера строк, а i делает поиск регистронезависимым.
  • |: Оператор конвейера (pipe), который передает вывод первой команды в качестве ввода для второй команды.
  • grep unsupp: Эта часть команды grep фильтрует вывод предыдущей команды, оставляя только строки, содержащие подстроку "unsupp".

Таким образом, после выполнения этой команды будут выведены все строки, содержащие слово "allow" из файлов в директории /usr/src/sys/dev/, где каждая строка будет снабжена номером строки, а затем из этого результата будут выбраны только те строки, которые также содержат подстроку "unsupp". Это может помочь в быстром нахождении и анализе соответствующих строк в коде исходного проекта.

И это была только вершина айсберга grep.
