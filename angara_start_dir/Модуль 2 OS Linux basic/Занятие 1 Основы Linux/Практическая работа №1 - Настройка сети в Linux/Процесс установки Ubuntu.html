Процесс установки Ubuntu

Тут Вы сможете ознакомиться с процессом создания виртуальной машинки на VMware и установкой Ubuntu.

Для начала создадим виртуальную машинку в VMware:

1.

2.

3. Можно выбрать 15 или 16 версию

4. Далее необходимо выбрать образ, который будем устанавливать

5. Заполним форму

6. Укажем имя под которым сохранится образ на нашем компьютере, далее укажем место куда необходимо сохранить

7. Укажем нужное количество процессоров, нам достаточно будет 2

8. Далее укажем нужное количество оперативной памяти, нам достаточно будет 2048 Мб

9. Тут выберем режим работы сетевого соединения

10. Выбор контроллера оставим по умолчанию

11. Режим работы виртуального жесткого диска, также оставим по умолчанию

12. Далее создадим новый жесткий диск для нашей Ubuntu

13. Назначем пространства для Ubuntu, для практического задания нам хватит 20 Gb

14. Далее нажмём Finish

15. Далее запустим вновь созданную виртуальную машинку и приступим к установке OS Ubuntu

16. Так выглядит процесс загрузки

17. Далее выберем язык операционной системы, рекомендую устанавливать Английскую версию, так как при конфигурировании русской версии встречается некорректная работа конфигурационных файлов, которые не рассчитаны на работу с кириллицей

18. Далее нажимаем кнопку продолжить по умолчанию, так как нам нет необходимости менять репозиторий и обновлять дистрибутив

Меню выглядит следующим образом:

1.Обновите до нового установщика

2.Продолжайте без обновления

3.Назад

19. Далее оставляем по умолчанию и нажимаем кнопку “Done”

20. Далее начинаем работу с сетевыми настройками, на нашем главном сетевом интерфейсе получаем IP адрес, по DHCP

21. Далее конфигурируем интерфейс “Bond0” после чего нажать кнопку “Create” и далее нажать “Done”

22. Настройку proxy оставляем по умолчанию

23. Тут адрес зеркала оставляем по умолчанию

24. Далее приступаем к настройке и выбора диска

Меню:

Настройте управляемый макет хранилища или создайте пользовательский:

  1. Используйте весь диск целиком - оставляем по умолчанию
  1. Настройте этот диск как группу LVM - если не выбрано, то надо выбрать
  1. Зашифруйте группу LVM с помощью LUKS - Существует множество других методов шифрования данных, но LUKS — лучший, поскольку он выполняет шифрование, работая на уровне ядра.
  1. Пользовательский макет хранилища - оставляем по умолчанию

24. Далее будет предложено назначить пароль, обязательно запомните или запишите его

25. Создаём пользователя и вписываем все строчки, запоминаем пароль

26. В этом окне менять ничего не надо, нажимаем “Done”

27. Тут по необходимости установка OpenSSH server

27. При необходимости можно выбрать какие утилиты нужно установить и нажать “Done”

28. Начнётся процесс установки

29. Далее будет предложена перезагрузка, можно перезапускать

30. После перезагрузки появится окно, где Ubuntu попросит ввести пароль от пункта где назначали шифрования

31. После чего загрузка пойдёт дальше и можно вводить пользователя и пароль от пользователя

32. Затем загрузка завершится, вы увидите следующее

33. Проверим, что у нас сконфигурировалось всё правильно, если всё хорошо, то Вы увидите следующее:

сетевые интерфейсы “lo, ens33 (в вашем случае может быть другое название), bond0”. В дальнейшем данные конфигурации нам будут нужны.

34. Далее переходим к пунктам 1.2 и 1.3 для понимания как работает процесс запуска операционной системы и что такое пользовательское окружение.
