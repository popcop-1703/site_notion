1.4.8 Утилита hping3

Цель работы:

  • Целью данной работы является оценка доступности удаленного сервера и анализ характеристик сетевого соединения с использованием утилиты hping3.

Задача:

  • Основной задачей является проверка доступности удаленного сервера с использованием ICMP-пакетов и анализ характеристик сетевого соединения, таких как задержка (RTT) и потеря пакетов.
  • Выполнить 3 сценария применения данной утилиты.
  • Составить отчёт о проделанной работе.
apt install hping3
hping3 -c 4 -q -1 <IP-адрес сервера>
  • c 4: Опция c указывает количество отправляемых пакетов (в данном случае, 4).
  • q: Опция q отключает вывод заголовков и информации о состоянии в stdout (делает вывод более компактным).
  • 1: Опция 1 указывает, что нужно использовать ICMP-пакеты для проверки доступности.

Вывод:

len=40 ip=<IP-адрес сервера> ttl=64 id=3967 icmp_seq=0 rtt=0.460 ms
len=40 ip=<IP-адрес сервера> ttl=64 id=3968 icmp_seq=1 rtt=0.412 ms
len=40 ip=<IP-адрес сервера> ttl=64 id=3969 icmp_seq=2 rtt=0.415 ms
len=40 ip=<IP-адрес сервера> ttl=64 id=3970 icmp_seq=3 rtt=0.423 ms
-- <IP-адрес сервера> hping statistic ---
4 packets transmitted, 4 packets received, 0% packet loss
round-trip min/avg/max = 0.412/0.425/0.460 ms

Описание вывода:

  1. len=40 ip=<IP-адрес сервера> ttl=64 id=3967 icmp_seq=0 rtt=0.460 ms:
    • len=40: Длина ICMP-пакета в байтах (в данном случае, 40 байт).
    • ip=<IP-адрес сервера>: IP-адрес сервера, на который отправляется пакет.
    • ttl=64: TTL (Time to Live) - количество "прыжков" пакета по сети, прежде чем он будет отброшен (в данном случае, 64).
    • id=3967: Идентификатор пакета.
    • icmp_seq=0: Последовательный номер ICMP-пакета (в данном случае, 0).
    • rtt=0.460 ms: Время в миллисекундах, затраченное на отправку пакета и получение ответа (round-trip time).
  1. Аналогичные строки следуют для остальных отправленных ICMP-пакетов, где меняется только icmp_seq и rtt для каждого пакета.
  1. -- <IP-адрес сервера> hping statistic ---:
    • Это разделитель, указывающий на начало статистической информации.
  1. 4 packets transmitted, 4 packets received, 0% packet loss:
    • Эта строка сообщает о количестве отправленных и полученных пакетах, а также о проценте потери пакетов. В данном случае, все 4 отправленных пакета были успешно получены, и процент потери равен 0%.
  1. round-trip min/avg/max = 0.412/0.425/0.460 ms:
    • Эта строка предоставляет статистику по времени задержки (RTT) для всех полученных пакетов. В данном случае, минимальное RTT составляет 0.412 мс, среднее - 0.425 мс, максимальное - 0.460 мс.

    1 сценарий: Создание нагрузки на сеть с помощью ICMP флуда.

    sudo hping3 -c 100 -i u10000 --icmp <IP-адрес сервера>

    Пример использования утилиты hping3 с подробным описанием вывода команды

    sudo hping3 -c 1000 -i u10000 --icmp <IP-адрес сервера>:

    • sudo hping3: Запускает утилиту hping3 с правами суперпользователя, так как для отправки ICMP-пакетов может потребоваться повышенные привилегии.
    • c 100: Опция c указывает количество отправляемых пакетов (в данном случае, 1000).
    • i u10000: Опция i задает интервальную задержку между отправкой пакетов. В данном случае, u1000 означает интервал в 1000 миллисекунд (10 секунд).
    • -icmp: Опция -icmp указывает, что будут использоваться ICMP-пакеты для проверки доступности.
    • <IP-адрес сервера>: Это IP-адрес удаленного сервера, который мы хотим протестировать.

    2 сценарий: Перегрузка сетевого адаптера с фрагментированными ICMP-пакетами.

    sudo hping3 -c 10000 --icmp --flood --rand-source 192.168.127.185

    sudo hping3 -c 10000 --icmp --flood --rand-source 192.168.127.185:

    • sudo hping3: Запускает утилиту hping3 с правами суперпользователя, так как для генерации флуда может потребоваться повышенные привилегии.
    • c 10000: Опция c указывает количество отправляемых пакетов (в данном случае, 10000).
    • -icmp: Опция -icmp указывает, что будут использоваться ICMP-пакеты для атаки.
    • -flood: Опция -flood указывает на режим флуда, в котором пакеты будут отправляться как можно быстрее.
    • -rand-source: Опция -rand-source указывает на использование случайных исходных IP-адресов отправителей.
    • 192.168.127.185: Это целевой IP-адрес сервера, который будет атакован.

    Вывод:

    • HPING 192.168.127.185 (ens33 192.168.127.2): icmp mode set, 28 headers + 0 data bytes: Это заголовок, который указывает на использование ICMP-пакетов и сетевого интерфейса, через который отправляются пакеты.
    • len=92 ip=192.168.127.185 ttl=64 icmp_seq=0 rtt=0.2 ms: Эта строка указывает, что был отправлен ICMP-пакет длиной 92 байта с указанными характеристиками. ip - IP-адрес целевого сервера, ttl - время жизни пакета, icmp_seq - последовательный номер пакета, rtt - время в миллисекундах, которое потребовалось для получения ответа от сервера.
    • len=92 ip=192.168.127.185 ttl=64 icmp_seq=1 rtt=0.2 ms: Аналогичная строка для второго ICMP-пакета.
    • len=92 ip=192.168.127.185 ttl=64 icmp_seq=2 rtt=0.2 ms: Аналогичная строка для третьего ICMP-пакета.
    • ... (аналогичные строки для остальных пакетов)
    • -- 192.168.127.185 hping statistic ---: Это заголовок для статистики.
    • 10000 packets transmitted, 10000 packets received, 0% packet loss: Это статистика по переданным и полученным пакетам. В данном случае, было отправлено 10000 пакетов и все 10000 успешно получены, что указывает на отсутствие потери пакетов.
    • round-trip min/avg/max = 0.2/0.2/0.3 ms: Эта строка содержит информацию о задержке (RTT) в миллисекундах для отправленных пакетов. Минимальное (min), среднее (avg) и максимальное (max) значения RTT предоставляют информацию о производительности сетевого соединения.

    Рекомендации:

    Составление отчета:

    • В отчете следует указать:
      • Цель работы (тестирование устойчивости и производительности сервера при атаке с флудом).
      • Методология (использованные опции hping3).
      • Результаты (статистика по отправленным и полученным пакетам, RTT).
      • Выводы (оценка устойчивости сервера и производительности в условиях атаки сетевым флудом).
      • Рекомендации (если есть, например, рекомендации по улучшению защиты сервера от подобных атак).

    3 сценарий: Проверка доступности порта на сервере

    $ hping3 -c 4 -S -p 22 192.168.127.185
    • sudo hping3: Запускает утилиту hping3 с правами суперпользователя, так как для отправки TCP-пакетов на низкие порты может потребоваться повышенные привилегии.
    • c 4: Опция c указывает количество отправляемых пакетов (в данном случае, 4).
    • S: Опция S указывает на использование флага SYN в TCP-пакетах, что означает начало установки TCP-соединения.
    • p 22: Опция p указывает на порт, на который будут отправлены TCP-пакеты (в данном случае, порт 22, который обычно используется для SSH-сервиса).
    • 192.168.127.185: Это IP-адрес целевого сервера, который будет протестирован.

    Вывод:

    • HPING 192.168.127.185 (ens33 192.168.127.2): S set, 40 headers + 0 data bytes: Это заголовок, который указывает на использование TCP-пакетов с установкой флага SYN и сетевого интерфейса, через который отправляются пакеты.
    • len=46 ip=192.168.127.185 ttl=64 DF id=0 sport=22 flags=SA seq=0 win=5840 rtt=0.3 ms: Эта строка указывает, что был отправлен TCP-пакет длиной 46 байт с указанными характеристиками. ip - IP-адрес целевого сервера, ttl - время жизни пакета, sport - исходный порт отправителя, flags - флаги TCP-пакета (в данном случае, SA означает, что это ответ на SYN-пакет), seq - последовательный номер пакета, win - размер окна, rtt - время в миллисекундах, которое потребовалось для получения ответа от сервера.
    • len=46 ip=192.168.127.185 ttl=64 DF id=0 sport=22 flags=SA seq=1 win=5840 rtt=0.3 ms: Аналогичная строка для второго TCP-пакета.
    • len=46 ip=192.168.127.185 ttl=64 DF id=0 sport=22 flags=SA seq=2 win=5840 rtt=0.3 ms: Аналогичная строка для третьего TCP-пакета.
    • ... (аналогичные строки для остальных пакетов)
    • -- 192.168.127.185 hping statistic ---: Это заголовок для статистики.
    • 4 packets transmitted, 4 packets received, 0% packet loss: Это статистика по переданным и полученным пакетам. В данном случае, было отправлено 4 TCP-пакета и все 4 успешно получены, что указывает на отсутствие потери пакетов.
    • round-trip min/avg/max = 0.3/0.3/0.3 ms: Эта строка содержит информацию о задержке (RTT) в миллисекундах для отправленных пакетов. Минимальное (min), среднее (avg) и максимальное (max) значения RTT предоставляют информацию о производительности сетевого соединения.

    Рекомендации:

    • Если все пакеты успешно установили соединение с сервером на указанном порту и не были потеряны, это указывает на доступность порта. Это положительное значение для проверки доступности конкретного порта на сервере.

    Составление отчета и рекомендаций:

    В отчете следует указать:

    • Цель работы (тестирование доступности SSH-сервиса на порту 22).
    • Методология (использованные опции hping3).
    • Результаты (статистика по отправленным и полученным пакетам, RTT).
    • Выводы (оценка доступности SSH-сервиса на порту 22).
    • Рекомендации (если есть, например, рекомендации по улучшению доступности SSH-сервиса).
