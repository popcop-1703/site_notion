2. Методические указания к выполнению

  • 2.1 Определение состояния сетевого подключения.
    • Просмотрите информацию по всем сетевым интерфейсам командой ifconfig –a
    • По выведенным данным определите Mac-адрес компьютера, ip-адрес компьютера, маску подсети, адрес широковещательной рассылки, адрес шлюза по
      умолчанию (если есть) и адрес dns-сервера (если есть).

    В процессе эксплуатации сети может возникнуть неисправность сети. Причиной неработоспособности сети может быть что угодно -начиная повреждения кабеля, проблем с драйвером сетевого адаптера и заканчивая параметром, который был упущен (частота Wi-Fi при ручной настройке, неверный пароль или что-либо подобное).

    Чаще всего в компьютерной сети используется получения IP-адреса по DHCP,
    однако иногда требуется и статическая адресация. Если автоматическая настройка не подходит, то необходимо настроить сетевые параметры.

    Выполните настройку сети двумя способами – через графический интерфейс и с командной строки.

    • Посмотреть ip-адреса выборочно (без лишней информации) можно с помощью следующих команд:
    /sbin/ip a | grep ‘inet’
    /sbin/ip a | grep ‘inet6’

  • 2.2 Как задать пароль пользователю root.
    • Запустите терминал - CTRL+ALT+T.
    • Задайте пароль пользователя root: sudo passwd root

  • 2.3 Настройка сетевых параметров через графический интерфейс.

    Из панели слева выберите:

    • SystemSettings (системные настройки)
    • Network (сетевые настройки). Можно видеть список различных видов сети.
    • Wired – проводная сеть.
    • Wireless – беспроводная сеть.

    Для того, чтобы настроить проводную сеть, выбираем необходимое подключение и для него выбираем «Options» «IPv4 Settings». Указываем сетевые настройки в соответствие со следующей таблицей:

    Адресс компьютера                    # 192.168.1.___ Например: 192.168.1.55
Маска подсети                        # 255.255.255.0
Основной шлюз                        # 192.168.1.1
Адреса DNS-серверов                  # 8.8.8.8
                                     # 4.4.4.4

    Убедитесь, что сетевой интерфейс удалось настроить, для этого введите команду ifconfig.

    Синтаксис команды ifconfig:

    ifconfig Название Интерфейса

    Пример:

    ifconfig eth0eth0 –от Ethernet0.

    Проверьте связь с соседними виртуальными машинами командой ping.

  • 2.4 Настройка сетевых параметров через консоль.

    Для этого:

    • Получите права администратора с помощью команды sudo su.
    • Перейдите в папку /etc/network - /etc/netplan/ Настройка Netplan
    • Отредактируйте файл interfaces Используйте редактор Nano.
    • Синтаксис: nano имя файла
    • В данном файле задаются настройки для всех сетевых интерфейсов компьютера.

    Терминология:

    • Loopback (коротко говоря loop или lo) — это программный метод, который
      направляет полученный сигнал или данные обратно отправителю.
    • В Windows: localhost (https://ru.wikipedia.org/wiki/Localhost). 
    auto lo                              #интерфейс для loopback интерфейса
iface lo inet loopback               #аналог Localhost

allow-hotplug eth1                   #автоматически выполнять перезапуск интерфейса при его падении
auto eth1                            #поднимать интерфейс автоматически при старте системы
#iface eth1 inet dhcp                #получить адрес по DHCP
iface eth1 inet static               #статический адрес
        address 192.168.1.1          #IP адрес
        netmask 255.255.255.0        #маска
        network 192.168.1.0          #задаём адрес сети
        broadcast 192.168.1.255      #задаём адрес для широковещательной рассылки
        gateway 192.168.0.254        #шлюз
        dns-nameserver 8.8.8.8       #задаём адреса DNS-серверов

    При отсутствии сетевого интерфейса eth0 добавьте его.

    Укажите сетевые параметры в соответствие с данной таблицей:

  • 2.6 Существует множество утилит в Linux для настройки сетевого интерфейса, но некоторые из них являются наиболее распространенными и полезными.
    1. ifconfig – утилита командной строки, которая используется для настройки сетевых интерфейсов, отображения текущих настроек интерфейса и мониторинга трафика.
    1. ip – более современная утилита Linux для настройки сетевых интерфейсов. Она является более функциональной и может выполнять действия, которые нельзя выполнить с помощью ifconfig.
    1. netstat – утилита, которая отображает информацию о текущих сетевых соединениях и состоянии портов.
    1. ethtool – утилита для настройки и управления сетевым оборудованием.
    1. route – утилита для настройки маршрутизации сети.
    1. dhclient – утилита для автоматической настройки сетевых параметров через DHCP.
    1. Network Manager – программное обеспечение, которое упрощает настройку сетевых интерфейсов в Linux. Оно позволяет управлять подключениями через графический интерфейс и автоматически подключаться к предпочитаемым сетям.

    Эти утилиты необходимы для настройки и управления сетевыми интерфейсами в Linux. Они помогают администраторам и пользователям настроить IP-адреса, маршрутизацию, DNS-серверы и другие параметры сетевого интерфейса.

  • 2.7 Как настроить DHCP в Linux.

    Для настройки DHCP в Linux используется утилита dhclient, которая позволяет автоматически получать и настраивать IP-адрес, маску подсети, шлюз и DNS-сервер из DHCP-сервера. Следуйте этим шагам для настройки DHCP в Linux:

    1. Установите пакет dhclient, если он ещё не установлен на вашей системе. В большинстве дистрибутивов Linux он уже установлен по умолчанию, но если необходимо, выполните следующую команду:
    sudo apt-get install dhclient
    1. Настройте сетевой интерфейс для работы с DHCP. Для этого вам нужно отредактировать конфигурационный файл сетевого интерфейса /etc/network/interfaces. Например, если вы хотите настроить сетевой интерфейс eth0:
    sudo nano /etc/network/interfaces

    Измените содержимое файла на следующее: 

    .# The primary network interface
auto eth0
iface eth0 inet dhcp
    1. Сохраните изменения и закройте файл.
    1. Запустите команду dhclient для запуска DHCP-клиента:
    sudo dhclient -v eth0
    1. После запуска команды dhclient система получит IP-адрес, маску подсети, шлюз и DNS-серверы от DHCP-сервера.Вы можете проверить настройки, выполнив команду следующей командой:
    ifconfig
    1. Чтобы сделать эти настройки постоянными, вы можете отредактировать файл /etc/network/interfaces следующим образом:
    .# The primary network interface
auto eth0
iface eth0 inet dhcp
    1. Сохраните изменения и закройте файл.

    Эти шаги настройки DHCP работают на большинстве дистрибутивов Linux. С помощью DHCP вы можете автоматически настраивать свой сетевой интерфейс без необходимости вручную настраивать IP-адреса, маршруты и DNS-серверы.

    1. Посмотреть активен ли dhclient или нет можно с помощью следующей команды:
    ps fax | grep dhclient

  • 2.8 Установки имени компьютера/домена.

    Команда hostname используется для просмотра имени компьютера и имени домена (DNS) (службы имен доменов), а также для изменения или установки имени компьютера/домена.

    hostname — это имя, которое присваивается компьютеру, подключенному к сети, которое однозначно идентифицирует в сети и, таким образом, позволяет получить доступ к нему без использования его IP-адреса.

    Основной синтаксис для команды hostname:

    hostname [options] [new_host_name]

    Если вы запустите команду hostname без каких-либо параметров, отобразится текущее имя хоста и доменное имя вашей системы Linux.

    Используя имя хоста, вы можете отобразить сетевой адрес (IP-адрес) имени хоста с флагом -i, а параметр -I устанавливает все настроенные сетевые интерфейсы и отображает все сетевые адреса хоста.

    И последнее, но не менее важное: изменение или установка имени хоста вашей системы Linux. Выполните следующую команду, не забудьте заменить «NEW_HOSTNAME» на фактическое имя хоста, которое вы хотите установить или изменить.

    sudo hostname NEW_HOSTNAME

    или

    sudo /etc/hostname
