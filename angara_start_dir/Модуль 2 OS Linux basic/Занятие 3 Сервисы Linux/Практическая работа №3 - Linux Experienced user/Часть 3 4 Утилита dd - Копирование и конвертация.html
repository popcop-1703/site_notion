Практическая работа 3.4.1: Копирование данных с одного диска на другой

Цель: Скопировать данные с одного диска на другой.

Задачи:

  1. Используйте dd для копирования данных с исходного диска /dev/sda на целевой диск /dev/sdb.

Решение:

sudo dd if=/dev/sda of=/dev/sdb bs=4M status=progress

Вывод:

  • sudo: это ключевое слово, которое предоставляет привилегии суперпользователя для выполнения следующей команды. Оно используется, так как операции с дисками часто требуют административных прав.
  • dd: утилита dd, как уже было упомянуто, используется для копирования и конвертации данных.
  • if=/dev/sda: это опция if (input file), которая указывает исходное устройство или файл, откуда будут считываться данные. В данном случае /dev/sda представляет исходный диск.
  • of=/dev/sdb: это опция of (output file), которая указывает целевое устройство или файл, куда будут записываться скопированные данные. Здесь /dev/sdb представляет целевой диск.
  • bs=4M: это опция bs (block size), которая задает размер блока данных, который будет считываться и записываться за одну операцию. В данном случае размер блока установлен на 4 мегабайта (4M). Изменение этого параметра может влиять на скорость копирования и потребление ресурсов.
  • status=progress: это опция status, которая добавляет вывод информации о текущем прогрессе выполнения операции. Она показывает текущее состояние операции, сколько данных уже было скопировано и скорость копирования.

Практическая работа 3.4.2: Создание образа диска

Цель: Создать образ диска для резервного копирования.

Задачи:

  1. Используйте dd для создания образа диска /dev/sda и сохраните его в файле disk_image.img.

Решение:

sudo dd if=/dev/sda of=disk_image.img bs=4M status=progress

Вывод:

  • sudo: это ключевое слово, которое предоставляет привилегии суперпользователя для выполнения команды. Поскольку операции с дисками часто требуют прав администратора, sudo используется для выполнения этой команды с административными правами.
  • dd: утилита dd используется для копирования и конвертации данных.
  • if=/dev/sda: опция if (input file) указывает исходное устройство или файл, откуда будут считываться данные. В данном случае /dev/sda представляет собой исходный диск, с которого будут скопированы данные.
  • of=disk_image.img: опция of (output file) указывает целевой файл, в который будут записаны скопированные данные. В данном случае, файл disk_image.img будет создан в текущем рабочем каталоге и будет содержать образ диска.
  • bs=4M: опция bs (block size) задает размер блока данных, который будет считываться и записываться за одну операцию. Здесь размер блока установлен на 4 мегабайта (4M). Изменение этого параметра может повлиять на скорость копирования и использование ресурсов.
  • status=progress: опция status добавляет вывод информации о текущем прогрессе выполнения операции. Это позволяет видеть текущее состояние операции, сколько данных уже скопировано и скорость копирования.

После выполнения этой команды, dd начнет копирование данных с диска /dev/sda и сохранит их в файл disk_image.img. В процессе выполнения операции будет выводиться информация о текущем прогрессе, и по завершении операции вы получите информацию о скорости и времени выполнения.

Образ диска будет сохранен в файле disk_image.img и может быть использован, например, для резервного копирования или восстановления данных с этого диска в будущем.

Практическая работа 3.4.3: Клонирование USB-флешки

Цель: Клонировать содержимое USB-флешки.

Задачи:

  1. Используйте dd для клонирования содержимого USB-флешки /dev/sdc на другую флешку /dev/sdd.

Для начала разберёмся как работать с флешь накопителем:

sudo lsblk

Команда sudo lsblk используется для вывода информации о блочных устройствах (включая диски и разделы) в системе. Если вы выполнили эту команду, она должна вывести список блочных устройств, их размеры, монтируемые точки и другую информацию.

Пример вывода команды sudo lsblk может выглядеть примерно так:

  • NAME - это имя устройства.
  • MAJ:MIN - это мажорный и минорный номера устройства.
  • RM - это флаг, указывающий, является ли устройство съемным (1 - да, 0 - нет).
  • SIZE - размер устройства.
  • RO - флаг, указывающий, доступно ли устройство только для чтения (1 - да, 0 - нет).
  • TYPE - тип устройства (disk - диск, part - раздел).
  • MOUNTPOINT - точка монтирования устройства (если оно смонтировано).

В выводе вы можете видеть информацию о дисках (sda, sdb), разделах (sda1, sda2, sda3, и так далее) и точках монтирования. Эта информация полезна для анализа состояния дисков и их использования в системе.

Команда df -h используется для вывода информации о доступном месте на файловых системах в системе. Опция -h делает вывод более читаемым для человека, представляя размеры в более удобных единицах (например, гигабайтах и мегабайтах, а не в блоках).

Команда sudo mount /dev/sdb1 /home/litvinigor/usb используется для монтирования (подключения) файловой системы, находящейся на устройстве /dev/sdb1, в каталог /home/litvinigor/usb.

В данной команде:

  • sudo: ключевое слово sudo используется для выполнения команды с правами суперпользователя (администратора), так как монтирование файловых систем обычно требует привилегий администратора.
  • mount: это сама команда для монтирования файловых систем.
  • /dev/sdb1: это исходное устройство, которое вы хотите монтировать. В данном случае, /dev/sdb1 представляет собой раздел или устройство, которое будет доступно в каталоге /home/litvinigor/usb.
  • /home/litvinigor/usb: это место, куда будет смонтирована файловая система. Файловая система, находящаяся на устройстве /dev/sdb1, будет доступна по этому пути.

После выполнения этой команды, файловая система с раздела /dev/sdb1 будет смонтирована в каталоге /home/litvinigor/usb, и вы сможете обращаться к данным на этой файловой системе через этот каталог.

Команда sudo ls /home/litvinigor/usb/ используется для отображения списка файлов и каталогов в директории /home/litvinigor/usb/, которая, как указали в предыдущей команде, является точкой монтирования для файловой системы, подключенной с устройства /dev/sdb1.

Команда sudo umount /home/litvinigor/usb используется для размонтирования (отключения) файловой системы, которая была ранее смонтирована в каталоге /home/litvinigor/usb. Это важно сделать перед тем, как извлечь устройство или флешку, чтобы избежать потери данных или повреждения файловой системы.

После выполнения этой команды, файловая система будет отключена от указанного каталога, и вы сможете безопасно извлечь устройство или флешку из системы.

Убедитесь, что у вас нет открытых файловых дескрипторов или процессов, которые могли бы использовать эту файловую систему, прежде чем выполнять команду umount, чтобы избежать потери данных или повреждения файлов.

Возвращаемся непосредственно к практической работе

Подготовим наши устройства для работы:

Решение:

sudo dd if=/dev/sdc of=/dev/sdd bs=4M status=progress

Команда sudo dd if=/dev/sdb1 of=/dev/sdc1 bs=4M status=progress используется для клонирования или копирования содержимого одного блочного устройства (/dev/sdb1) на другое блочное устройство (/dev/sdc1) с использованием утилиты dd в Linux. Давайте разберем каждую часть этой команды:

  • sudo: это ключевое слово, которое используется для выполнения следующей команды с привилегиями суперпользователя, так как операции с блочными устройствами обычно требуют административных прав.
  • dd: утилита dd используется для копирования и конвертации данных.
  • if=/dev/sdb1: это опция if (input file), которая указывает исходное блочное устройство, с которого будут считываться данные. В данном случае /dev/sdb1 представляет исходное устройство.
  • of=/dev/sdc1: это опция of (output file), которая указывает целевое блочное устройство, куда будут записываться скопированные данные. Здесь /dev/sdc1 представляет целевое устройство.
  • bs=4M: это опция bs (block size), которая задает размер блока данных, который будет считываться и записываться за одну операцию. В данном случае размер блока установлен на 4 мегабайта (4M). Изменение этого параметра может влиять на скорость копирования и потребление ресурсов.
  • status=progress: это опция status, которая добавляет вывод информации о текущем прогрессе выполнения операции. Она показывает текущее состояние операции, сколько данных уже было скопировано и скорость копирования.

После выполнения этой команды, dd начнет копирование данных с блочного устройства /dev/sdb1 на блочное устройство /dev/sdc1. В выводе команды будет отображаться текущий прогресс операции, и по завершении копирования вы увидите информацию о скорости и времени выполнения.

Пожалуйста, будьте очень осторожны при использовании dd для подобных операций, так как неправильное использование этой утилиты может привести к потере данных. Убедитесь, что вы правильно выбрали исходное и целевое блочные устройства перед выполнением подобных команд.

Практическая работа 4: Запись образа на диск или флешку

Цель: Записать ранее созданный образ диска на новый диск или флешку.

Задачи:

  1. Используйте dd для записи образа диска disk_image.img на новый диск /dev/sdс.

Решение:

Первая Ubuntu:

Мы делаем дамп или копию/образ раздела sda3, который зашифрован при установке.

sudo dd if=/dev/sda3 of=/home/litvinigor/data/sda3.img bs=4M status=progress

Команда sudo dd if=/dev/sda3 of=/home/litvinigor/data/sda3.img bs=4M status=progress создает образ раздела /dev/sda3 и сохраняет его в файл /home/litvinigor/data/sda3.img. В данном случае, образ будет создан с блочным размером 4 мегабайта и будет отображать прогресс операции.

По завершении этой команды, вы получите файл sda3.img, который содержит точную копию содержимого раздела /dev/sda3. Вы сможете использовать этот образ для резервного копирования данных или для восстановления раздела, если это будет необходимо.

sudo dd if=/home/litvinigor/data/sda3.img of=/dev/sdc bs=4M status=progress
20396900352 bytes (20 GB, 19 GiB) copied, 1405 s, 14.5 MB/s
4863+1 records in
4863+1 records out
20397948928 bytes (20 GB, 19 GiB) copied, 1460.66 s, 14.0 MB/s

Команда sudo dd if=/home/litvinigor/data/sda3.img of=/dev/sdc bs=4M status=progress выполняет обратную операцию и записывает содержимое образа раздела /home/litvinigor/data/sda3.img на устройство /dev/sdc. Это позволит восстановить содержимое раздела /dev/sda3 на устройстве /dev/sdc.

При выполнении этой команды образ будет записан на выбранное устройство блочными порциями размером 4 мегабайта, и вы увидите прогресс операции. После завершения команды, данные из образа будут восстановлены на устройство /dev/sdc. Будьте осторожны, так как это перезапишет данные на /dev/sdc, и все существующие данные на этом устройстве будут утеряны. Убедитесь, что выбрали правильное устройство для записи.

Вторая Ubuntu:

На другой системе необходимо выполнить:

sudo cryptsetup luksOpen /dev/sdc my-dm_crypt-1

Команда sudo cryptsetup luksOpen /dev/sdc my-dm_crypt-1 открывает устройство /dev/sdc с помощью LUKS шифрования и называет его my-dm_crypt-1. После успешного выполнения этой команды, устройство будет доступно для использования в виде /dev/mapper/my-dm_crypt-1. Вы можете использовать это устройство для монтирования файловых систем или выполнения других операций, требующих доступа к данным на зашифрованном диске.

igor@ubuntu:~$ sudo apt install vgchange

Команда vgchange не является отдельным пакетом для установки. Эта команда входит в состав утилиты LVM (Logical Volume Management), которая обычно уже установлена в большинстве дистрибутивов Linux, включая Ubuntu.

Чтобы использовать команду vgchange, вам не нужно устанавливать ее отдельно. Вы можете выполнять эту команду непосредственно в терминале после установки системы. Если вы столкнулись с проблемами при выполнении этой команды, убедитесь, что у вас установлены пакеты LVM. Вы можете убедиться, что они установлены, выполнив следующую команду:

sudo apt install lvm2 - может не сработать

Тогда:

igor@ubuntu:~$ sudo rm /var/cache/apt/archives/lock
igor@ubuntu:~$ sudo rm /var/lib/dpkg/lock-frontend

Удаление файла /var/cache/apt/archives/lock может привести к разблокировке кеша APT, но это не является рекомендуемым способом устранения блокировки. Лучше использовать следующий способ:

  1. Убедитесь, что нет активных процессов APT. Вы можете проверить это с помощью следующей команды: 
    ps aux | grep -i apt

    Если вы видите процессы apt или dpkg, дождитесь их завершения.

  1. Если все процессы APT завершены, но блокировка осталась, удалите файл блокировки следующей командой: 
    sudo rm /var/lib/dpkg/lock-frontend

Повторите команды:

igor@ubuntu:~$ sudo apt-get update
igor@ubuntu:~$ sudo apt-get install lvm2

igor@ubuntu:~$ sudo vgscan
igor@ubuntu:~$ sudo vgchange -ay

  1. sudo vgscan:
    • Эта команда сканирует систему на предмет групп томов (VG) с использованием метаданных LVM2.
    • Когда вы выполнили эту команду, она обнаружила группу томов с именем "ubuntu-vg", использующую метаданные типа LVM2 в вашей системе.
  1. sudo vgchange -ay:
    • Эта команда активирует (или включает) группу томов, указанную в конфигурации LVM.
    • В вашем случае она активировала группу томов "ubuntu-vg".
    • Вывод "1 logical volume(s) in volume group 'ubuntu-vg' now active" указывает на то, что один логический том внутри группы томов "ubuntu-vg" был активирован и теперь доступен для использования.

В итоге, вы успешно выполнили сканирование и активацию группы томов "ubuntu-vg", что позволило сделать ее логические тома доступными и готовыми к использованию в вашей системе.

sudo mount /dev/ubuntu-vg/ubuntu-lv /home/igor/usb

Вы попытались примонтировать логический том /dev/ubuntu-vg/ubuntu-lv в директорию /home/igor/usb. Если вы не получили сообщений об ошибках, то монтирование должно было выполниться успешно. Ваш логический том теперь доступен по пути /home/igor/usb. Вы можете просматривать его содержимое и работать с данными в этой директории

ls /home/igor/usb

sudo umount /dev/ubuntu-vg/ubuntu-lv /home/igor/usb

Команда sudo umount /dev/ubuntu-vg/ubuntu-lv /home/igor/usb выполняет отмонтирование двух разделов:

  1. /dev/ubuntu-vg/ubuntu-lv: Это логический раздел внутри тома LVM (Logical Volume Management) с именем "ubuntu-vg". LVM предоставляет гибкую систему управления разделами. Эта команда отмонтирует логический раздел "ubuntu-lv", который, вероятно, был предварительно примонтирован.
  1. /home/igor/usb: Это каталог или точка монтирования, в которую мог быть предварительно примонтирован файловый раздел. Команда также отмонтирует этот раздел.

Обратите внимание, что эта команда требует прав администратора (через sudo), поскольку отмонтирование разделов обычно требует привилегий суперпользователя.

Чтобы отключить (деактивировать) том с помощью vgchange, вы можете использовать опцию -an, которая означает "отключить все тома". 

sudo vgchange -an <имя-тома>

Замените <имя-тома> на имя вашего LVM-тома, который вы хотите отключить. Например:

sudo vgchange -an ubuntu-vg

После выполнения этой команды том будет отключен и не будет доступен для использования до тех пор, пока его не активируют снова с помощью vgchange -ay.

Команда sudo cryptsetup close my-dm_crypt-1 предполагает закрытие (отключение) LUKS-контейнера с именем my-dm_crypt-1.

Для успешного закрытия LUKS-контейнера, сначала убедитесь, что контейнер был успешно открыт. Вы можете использовать команду sudo cryptsetup luksOpen /dev/sdX my-dm_crypt-1, где /dev/sdX - это ваш LUKS-контейнер, а my-dm_crypt-1 - имя, которое вы хотите присвоить открытому устройству. После успешного открытия контейнера вы сможете закрыть его с помощью sudo cryptsetup close my-dm_crypt-1.

Нажмите Tab команды sudo cryptsetup, и вам отобразился список доступных подкоманд и операций, которые можно выполнять с LUKS (Linux Unified Key Setup) контейнерами.

  1. benchmark: эта команда позволяет выполнить бенчмарк (тест производительности) на LUKS-контейнере.
  1. close: закрыть (отключить) LUKS-контейнер.
  1. luksChangeKey: изменить ключ (пароль) для LUKS-контейнера.
  1. luksDump: вывести информацию о LUKS-контейнере, включая его параметры и заголовок.
  1. luksFormat: форматировать устройство для использования LUKS-контейнером.
  1. luksRemoveKey: удалить ключ (пароль) из LUKS-контейнера.
  1. open: открыть (подключить) LUKS-контейнер для доступа к данным.
  1. status: отобразить текущий статус LUKS-контейнера.
  1. repair: попытаться восстановить поврежденный LUKS-контейнер.
  1. resize: изменить размер LUKS-контейнера.
  1. isLuks: проверить, является ли устройство LUKS-контейнером.
  1. erase: стереть данные из LUKS-контейнера (предостережение: это безвозвратно удалит данные).
  1. tcryptDump: вывести информацию о TCRYPT-контейнере (TrueCrypt-compatible), если поддерживается.

Вы можете выбрать необходимую подкоманду, а затем выполнить соответствующую операцию с вашим LUKS-контейнером. Например, для открытия контейнера используйте команду sudo cryptsetup open /dev/sdX mycontainer, где /dev/sdX - это ваш LUKS-контейнер, а mycontainer - имя, которое вы выбрали для открытого устройства.

Доступные подкоманды sudo cryptsetup luks:

  1. luksAddKey: добавление нового ключа шифрования к LUKS-зашифрованному устройству.
  1. luksChangeKey: изменение существующего ключа шифрования.
  1. luksDump: вывод информации о текущем состоянии LUKS-зашифрованного устройства, такой как количество слотов и используемые ключи.
  1. luksFormat: форматирование устройства как LUKS-зашифрованное устройство и создание первичного ключа.
  1. luksHeaderBackup: создание резервной копии заголовка LUKS в файле.
  1. luksHeaderRestore: восстановление заголовка LUKS из резервной копии.
  1. luksKillSlot: удаление ключа шифрования из указанного слота.
  1. luksRemoveKey: удаление ключа шифрования.
  1. luksResume: возобновление работы системы после приостановки (предназначено для систем с гибернацией).
  1. luksSuspend: приостановка работы системы (предназначено для систем с гибернацией).

Вы можете использовать эти подкоманды для управления ключами шифрования и выполнения других операций с LUKS-зашифрованными устройствами.
