1. Методические указания к выполнению

1.1 Поиск файлов

Предположим, что вы только что установили новую версию Ubuntu на ваш компьютер собираетесь добавить к Python некоторые скрипты. Вы проводите поиск в Интернете, и видите, что используются две разные версии Python, и вы не знаете, какая из них был установлен в вашей системе установщиком Ubuntu, и устанавливал ли он какие-либо дополнительные модули. Чтобы с этим разобраться, просто запустите эту команду:

dpkg -l | grep -i python
Команда dpkg -l | grep -i python выполняет поиск в списке всех установленных пакетов (dpkg -l) строк, содержащих фразу "python" (с учетом регистра, благодаря опции -i в grep). В операционных системах на базе Debian и их производных, таких как Ubuntu, dpkg является инструментом для управления пакетами. В данном контексте, dpkg -l отображает список всех установленных пакетов на вашей системе.

Пример вывода:

  • ii python 2.7 2.7.3-0ubuntu3.4 Interactive high-level object-oriented language (version 2.7)
  • ii python2.7-minimal 2.7.3-0ubuntu3.4 Minimal subset of the Python language (version 2.7)
  • ii python-openssl 0.12-1ubuntu2.1 Python wrapper around the OpenSSL library
  • ii python-pam 0.4.2-12.2ubuntu4 A Python interface to the PAM library

Сначала мы запустили dpkg -l, чтобы вывести установленные *.deb пакеты в вашей системе. Далее мы передали этот вывод в grep -i python. Опция -i используется, чтобы игнорировать регистр, поскольку grep чувствителен к нему. Использование опции -i — хорошая привычка, если вы конечно, не пытаетесь выполнить более конкретный поиск.

1.2 Поиск и фильтрация файлов

Grep также может использоваться для поиска и фильтрации внутри одного или нескольких файлов. Давайте рассмотрим эту функцию:

Допустим, у вас возникли проблемы с вашим веб-сервером Apache, и вы обратились к одному из многих форумов в сети с просьбой о помощи. Добрая душа, которая вам ответила, попросила вас отправить содержимое вашего файла /etc/hosts. Разве вам не было бы не легче, если бы вы могли просто удалить все закомментированные строки? А ведь вы это можете! Просто выполните эту команду:

grep -v '#' /etc/hosts
Команда grep -v '#' /etc/hosts используется для поиска в файле /etc/hosts строк, которые не начинаются с символа #. В операционных системах UNIX и их производных, таких как Linux, файл /etc/hosts используется для соответствия IP-адресов именам хостов.
После выполнения этой команды, вы получите результат, который представляет собой содержимое файла /etc/hosts, за исключением строк, начинающихся с символа #. Символ # обычно используется для обозначения комментариев в конфигурационных файлах, поэтому использование -v в grep исключает строки с комментариями, оставляя только строки, содержащие конфигурационные записи.

Опция -v указывает grep инвертировать свой вывод, что означает, что вместо печати совпадающих строк grep сделает противоположное и выведутся строки, которые не соответствуют выражению, в этом случае – закомментированные строки.

2. Найти все файлы .mp3

Grep может быть очень полезен для фильтрации из stdout. Например, предположим, что у вас есть целая папка, полная музыкальных файлов разных форматов. Вы хотите найти все файлы *.mp3 у исполнителя ABC, но вы не хотите никаких ремиксов. Использование команды find с несколькими опциями grep запросто проделает этот трюк:

find . –name "*.mp3" | grep –i ABC | grep –vi "remix"

Первый фильтр: find . -name "*.mp3"

Это команда find, которая начинает поиск файлов в текущем каталоге (.) и его подкаталогах. Опция -name "*.mp3" говорит find искать файлы только с расширением .mp3.

grep -i ABC

Этот фильтр grep ищет строки, содержащие фразу "ABC" в имени файла. Опция -i делает поиск регистронезависимым, то есть игнорирует регистр символов.

grep -vi "remix”
Этот фильтр
grep исключает строки, содержащие слово "remix". Опция -v отображает только строки, которые не содержат "remix", а -i делает поиск регистронезависимым.

В этом примере мы используем find для вывода всех файлов с расширением *.mp3, связывая поиск с grep -i, чтобы отфильтровать и вывести все файлы с именем «ABC», а затем еще одна опция для grep -vi, которая фильтрует и не выводит все имена файлов со словом «remix».

3. Отображение количества строк до или после строки поиска

Еще пара опций — это функции -A и -B, которые отображают согласованные строки и количество строк, которые присутствуют до или после строки поиска. В то время как справочная страница дает более подробное объяснение, нам легче запомнить параметры как -A — after и -B — before:

ifconfig | grep -A 4 eth0
После выполнения команды ifconfig, вы получите подробную информацию о сетевых интерфейсах, включая их IP-адреса, состояние подключения, MAC-адреса и другие параметры.
Однако команда grep -A 4 eth0 применяется к выводу ifconfig и используется для фильтрации результатов:
  • grep: Это утилита для поиска текста в выводе команд.
  • A 4: Опция A указывает, что после каждого совпадения выводится указанное количество строк после строки совпадения. В данном случае 4 означает, что после каждого совпадения будет выведено 4 строки.
  • eth0: Это аргумент, который вы ищете в выводе ifconfig. В данном случае, скорее всего, это имя сетевого интерфейса Ethernet.
ifconfig | grep -B 2 UP
Команда grep -B 2 UP применяется к выводу ifconfig и используется для фильтрации результатов:
  • grep: Это утилита для поиска текста в выводе команд.
  • B 2: Опция B указывает, что перед каждым совпадением выводятся указанные строки перед строкой совпадения. В данном случае 2 означает, что перед каждым совпадением будет выведено 2 строки.
  • UP: Это аргумент, который вы ищете в выводе ifconfig. В контексте сетевых интерфейсов "UP" обычно указывает на то, что интерфейс включен и активен.

4. Вывод количества строк совпадения

Параметр -C для grep вместо того, чтобы проверять строки, которые появляются до или после искомой строки, он проверяет строки в любом направлении:

ifconfig | grep -C 2 lo
Команда grep -C 2 lo применяется к выводу ifconfig и используется для фильтрации результатов:
  • grep: Это утилита для поиска текста в выводе команд.
  • C 2: Опция C указывает, что вместе с каждым совпадением выводится указанное количество строк как перед совпадением, так и после него. В данном случае 2 означает, что перед и после каждого совпадения будет выведено по 2 строки.
  • lo: Это аргумент, который вы ищете в выводе ifconfig. lo - это сетевой интерфейс "loopback", который используется для обратной связи на локальном устройстве.

5. Количество совпадений

Grep так же может запросто подсчитать количество совпадений:

ifconfig | grep -c inet6
Команда grep -c inet6 применяется к выводу ifconfig и используется для подсчета количества строк, содержащих строку inet6. В сетевых контекстах inet6 обычно относится к адресам IPv6.
  • grep: Это утилита для поиска текста в выводе команд.
  • c: Опция c используется для вывода только количества строк, соответствующих критериям поиска, а не самих строк.

6. Поиск файлов по заданным номерам строк

Параметр -n для grep очень полезен при отладке файлов во время компиляции. Он отображает номер строки в файле поиска:

grep -n 'main' setup.py
Команда grep -n 'main' setup.py используется для поиска всех строк в файле setup.py, содержащих слово 'main', и вывода их с номерами строк.
  • grep: Это утилита для поиска текста в файле.
  • n: Опция n используется для отображения номеров строк вместе с результатами поиска.
  • 'main': Это шаблон, который grep ищет в файле setup.py. В данном случае он ищет строки, содержащие слово 'main'.

7. Поиск рекурсивной строки во всех каталогах

Если вы хотите найти строку в текущем каталоге вместе со всеми подкаталогами, вы можете указать параметр -r для поиска рекурсивно:

grep –r "function" *
Команда grep -r "function" * выполняет поиск всех вхождений строки "function" в содержимом файлов в текущем каталоге и его подкаталогах.
  • grep: Это утилита для поиска текста в файле или наборе файлов.
  • r: Опция r (или -recursive) указывает grepу рекурсивно просматривать все подкаталоги.
  • "function": Это строка, которую grep ищет в файлах.
  • : Это маска, которая соответствует всем файлам и подкаталогам в текущем каталоге.

8. Поиск по всему шаблону

Опции -w для grep ищет весь шаблон, который находится в строке. Например, используя:

ifconfig | grep -w "RUNNING"
Команда grep -w "RUNNING" применяется к выводу ifconfig и используется для фильтрации результатов:
  • grep: Это утилита для поиска текста в выводе команд.
  • w: Опция w в grep ищет только целые слова. Это означает, что grep будет искать только полные совпадения с словом "RUNNING", игнорируя части слов или подстроки.

Выведет строку, содержащую шаблон в кавычках. С другой стороны, если вы попытаетесь:

ifconfig | grep -w 'RUN'
Команда grep -w 'RUN' применяется к выводу ifconfig и используется для фильтрации результатов:
  • grep: Это утилита для поиска текста в выводе команд.
  • w: Опция w в grep ищет только целые слова. Это означает, что grep будет искать только полные совпадения с словом "RUN", игнорируя части слов или подстроки.

Ничего не будет возвращено, поскольку мы ищем не шаблон, а целое слово.

9. Поиск строки в архивах файлах Gzip

Gzip (сокращение от GNU Zip) — утилита сжатия и восстановления (декомпрессии) файлов, использующая алгоритм Deflate. Для поиска в таких файлах необходимо использовать команду zgrep, которая принимает те же параметры, что и grep, и используется таким же образом:

zgrep -i error /usr/share/info/wget.info.gz
Команда zgrep -i error /usr/share/info/wget.info.gz используется для поиска строк, содержащих слово "error" в файле wget.info.gz, который является сжатым файлом информации о программе wget.
  • zgrep: Это утилита, аналогичная grep, но способная работать с сжатыми файлами. Она ищет текст в файлах, сжатых с использованием gzip.
  • i: Это опция zgrep, которая указывает на выполнение поиска без учета регистра символов. То есть поиск будет нечувствителен к регистру, и строки, содержащие "Error", "ERROR" и "error", будут рассматриваться как совпадения.
  • error: Это строка, которую zgrep ищет в файле wget.info.gz.

9.1 Создаем простой zip-архив

Для создания zip-архива просто выполняем команду zip, в первом аргументе указываем имя будущего архива, а во втором сам файл, который мы сжимаем:

zip myfile.zip myfile.txt
Команда zip myfile.zip myfile.txt используется для создания нового архива с именем myfile.zip, в который включается файл myfile.txt.
  • zip: Это утилита для архивации файлов и каталогов.
  • myfile.zip: Это имя архива, который вы создаете.
  • myfile.txt: Это файл, который вы добавляете в архив.

Если нужно сжать несколько файлов то перечисляем их через пробел:

zip myarchive.zip myfile.txt yourfile.txt theirfile.txt
Команда zip myarchive.zip myfile.txt yourfile.txt theirfile.txt используется для создания нового архива с именем myarchive.zip, в который включены файлы myfile.txt, yourfile.txt и theirfile.txt.
  • zip: Это утилита для архивации файлов и каталогов.
  • myarchive.zip: Это имя архива, который вы создаете.
  • myfile.txt, yourfile.txt, theirfile.txt: Это файлы, которые вы добавляете в архив.

Таким образом, после выполнения этой команды файлы myfile.txt, yourfile.txt и theirfile.txt будут добавлены в архив с именем myarchive.zip. Получившийся архив будет содержать все эти файлы. Создание архива позволяет объединить несколько файлов в один для удобства хранения, передачи или архивации.

Создаем zip-архив папки, чтобы заархивировать папку, используется ключ -r:

zip -r mydir.zip verygooddir
Команда zip -r mydir.zip verygooddir используется для создания нового архива с именем mydir.zip, который содержит все файлы и подкаталоги в каталоге verygooddir, а также все файлы и подкаталоги внутри этих подкаталогов.
  • zip: Это утилита для архивации файлов и каталогов.
  • r: Опция r (или -recursive) указывает zipу рекурсивно включить в архив все файлы и подкаталоги в указанном каталоге и его подкаталогах.
  • mydir.zip: Это имя архива, который вы создаете.
  • verygooddir: Это имя каталога, содержимое которого вы хотите добавить в архив.

Создаем zip-архив с паролем

Очень важной функцией утилиты zip является возможность задания пароля на распаковку архива. Для этого применяется опция -P, после которой следует написать пароль:

zip -P мойпароль -r mysecretdir.zip mysecretdir
Команда zip -P мойпароль -r mysecretdir.zip mysecretdir используется для создания зашифрованного архива с именем mysecretdir.zip, который содержит все файлы и подкаталоги в каталоге mysecretdir, а также все файлы и подкаталоги внутри этих подкаталогов. Архив будет зашифрован с помощью пароля мойпароль.
  • zip: Это утилита для архивации файлов и каталогов.
  • P мойпароль: Это опция P, которая используется для указания пароля для зашифровки архива. мойпароль здесь представляет ваш пароль.
  • r: Опция r (или -recursive) указывает zipу рекурсивно включить в архив все файлы и подкаталоги в указанном каталоге и его подкаталогах.
  • mysecretdir.zip: Это имя архива, который вы создаете.
  • mysecretdir: Это имя каталога, содержимое которого вы хотите добавить в архив.

Если вы не хотите вводить пароль в командной строке у всех на виду, то можно использовать опцию -e, чтобы вместо ввода пароля в открытую, вводить его в срытом виде:

zip -er mysecretdir.zip mysecretdir
Команда zip -er mysecretdir.zip mysecretdir используется для создания зашифрованного архива с именем mysecretdir.zip, который содержит все файлы и подкаталоги в каталоге mysecretdir.
  • zip: Это утилита для архивации файлов и каталогов.
  • e: Опция e (или -encrypt) используется для указания на необходимость зашифровки создаваемого архива. При использовании этой опции zip запросит у вас пароль для шифрования архива.
  • r: Опция r (или -recursive) указывает zipу рекурсивно включить в архив все файлы и подкаталоги в указанном каталоге и его подкаталогах.
  • mysecretdir.zip: Это имя архива, который вы создаете.
  • mysecretdir: Это имя каталога, содержимое которого вы хотите добавить в архив.

После выполнения данной команды, вам будет предложено дважды ввести пароль. Сам пароль виден при этом не будет:

Enter password:
Verify password:

Распаковка zip-архива

Для того, чтобы разархивировать zip-архив, используется команда unzip. Ее можно запускать без опций, только указывая имя архива:

unzip myfile.zip
Команда unzip myfile.zip используется для распаковки архива с именем myfile.zip.
  • unzip: Это утилита для извлечения файлов из архивов.
  • myfile.zip: Это имя архива, который вы хотите распаковать.

После выполнения этой команды содержимое архива myfile.zip будет извлечено и разархивировано в текущий рабочий каталог. Если в архиве есть подкаталоги, то они будут созданы в текущем каталоге, а файлы будут извлечены в соответствующие подкаталоги.

По умолчанию распаковка происходит в текущей директории. Чтобы распаковать архив в другую директорию, используется опция -d, после которой нужно указать путь до директории:

unzip myfile.zip -d mydir
Команда unzip myfile.zip -d mydir используется для распаковки архива с именем myfile.zip и извлечения его содержимого в указанный каталог mydir.
  • unzip: Это утилита для извлечения файлов из архивов.
  • myfile.zip: Это имя архива, который вы хотите распаковать.
  • d mydir: Опция d используется для указания каталога, в который вы хотите извлечь содержимое архива. mydir здесь представляет имя целевого каталога.
