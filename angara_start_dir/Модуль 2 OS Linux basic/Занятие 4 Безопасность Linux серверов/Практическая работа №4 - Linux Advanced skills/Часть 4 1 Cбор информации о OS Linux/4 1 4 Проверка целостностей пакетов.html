4.1 Теория

4.1.1 dpkg

dpkg является довольно низкоуровневой утилитой. Существуют более высокоуровневые утилиты, например APT, которые могут загружать пакеты из сетевого репозитория и отслеживать зависимости между пакетами. Конечным пользователям следует использовать утилиты с более дружественным интерфейсом, такие как Aptitude или Synaptic, предоставляющие лёгкий способ просмотра списка пакетов, их описания и зависимостей.

Основные функции:

  • Установка deb пакета: dpkg -i имя_файла.deb
  • Удаление deb пакета: dpkg -r имя_пакета
  • Удаление deb пакета вместе с конфигурационными файлами: dpkg -P имя_пакета
  • Посмотреть расположение файлов установленного deb пакета: dpkg -S имя_пакета
  • Посмотреть будущее расположение файлов и каталогов неустановленного deb пакета dpkg -c имя_файла.deb
  • Для получения информации о deb пакете воспользуйтесь командой: dpkg -I имя_файла.deb
  • Просмотреть информацию о всех установленных deb пакетов: dpkg -l

4.1.2 debsums

debsums служит, в первую очередь, для нахождения изменённых администратором файлов установленных в системе пакетов или повреждённых файлов из-за некачественного носителя и в какой-то мере используется в качестве инструмента обеспечения безопасности.

  • debsums -a - Проверяет все файлы (включая конфигурационные, которые обычно располагаются слева)
  • debsums -e - Проверяет только конфигурационные файлы пакетов
  • debsums -с - Выводит список измененных на stdout файлов
  • debsums -I - Отображает файлы, в которых отсутствует информация о md5sum
  • debsums -s - Выводит только ошибки; при их отсутствии не производит никаких действий

debsums <название программного пакета> - Отображает пакеты, которые необходимо проверить с помощью debsums

4.2 Создание инцидента

4.2.1 Просмотр пакетов

Просмотр всех утилит:

dpkg -l

Просмотр установленных утилит, с помощью этой команды, можно увидеть все штатные утилиты:

dpkg -L coreutils

4.2.2 Выбираем утилиту

Для примера рассмотрим утилиту /bin/dir, которая позволяет нам работать по сути как ls

/bin/dir --help

4.2.3 Подмена

Что будет если мы поменяем /bin/false на /bin/dir?

Тем самым мы имитируем подмену или правку кода программы.

  1. Для начала сделаем бэкап команды:
sudo mv /bin/dir /bin/dir.bak
  1. Далее скопируем false в dir

Вводим команду

sudo cp /bin/false /bin/dir
  1. Теперь при отправке команды dir мы можем заметить, что ничего не будет происходить. Проверим, успешно ли была выполнена предыдущая команда. В качестве вывода замечаем логическую единицу (это указывает на ошибку в выполнении):
echo $?

4.3 Расследование инцидента

4.3.2 Проверка dpkg

Проверим штатные утилиты на несоответствие

dpkg --verify coreutils

Проверка нам выдала тот самый /bin/dir.

4.3.2 Проверка debsums

Чтобы нам быстро проверить этот пакет мы можем выполнить команду

debsums

и если она не установлена, то необходимо её установить через команду

sudo apt install debsums

Теперь запустим проверки

  • debsums -ac coreutils - запуск проверки
  • debsums -cs coreutils - запуск проверки

Проверка тоже нам выдала тот самый подмененный /bin/dir.

4.4 Проверка системы с помощью find

Если кто-то смог проникнуть в систему и что-то делал в ней, нам поможет команда find.

Например, если нам надо рассмотреть какой либо инцидент:

sudo find / -printf "%T@;%Tc;%p\n"

можно указать через пробел, но тогда будет сложно работать с выводом данных:

sudo find / -printf "%T@ %Tc %p\n"

Далее если надо сделать по одному конкретному пользователю необходимо следующее:

sudo find /home/kali/ -printf "%T@;%Tc;%p\n" > timeline.csv

Затем желательно это отсортировать, например:

head timeline.csv сейчас никак не отсортировано

Но давайте отсортируем данный файл с помощью команды:

sort -rn timeline.csv | less

где флаг -r как раз будет указывать, что последнее изменение будет первым

Мы можем видеть, что конкретно менялось и в какое время. Это позволит увидеть нам все изменения.
