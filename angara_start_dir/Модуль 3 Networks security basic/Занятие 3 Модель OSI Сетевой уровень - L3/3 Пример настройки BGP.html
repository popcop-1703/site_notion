Протокол BGP

Настройка IP адресов.

Для того, что бы показать как работает BGP настроим C-iBGP-102, M-iBGP-101 и C-iBGP-23.

Настройка на C-iBGP-102:

Router>en
Router#conf t
Router(config)#hostname C-iBGP-102
C-iBGP-102(config)#int range e0/2-3, e1/0
C-iBGP-102(config-if-range)#no shutdown
C-iBGP-102(config-if-range)#int e0/2
C-iBGP-102(config-if)#ip addr 13.1.1.102 255.255.255.0
C-iBGP-102(config-if)#int e0/3
C-iBGP-102(config-if)#ip addr 13.1.2.102 255.255.255.0
C-iBGP-102(config-if)#int e1/0
C-iBGP-102(config-if)#ip addr 13.1.3.102 255.255.255.0
C-iBGP-102(config-if)#do wr

Настройка на M-iBGP-101:

[admin@MikroTik] >
[admin@MikroTik] /system identity> set name=M-iBGP-101
[admin@M-iBGP-101] /system identity> print
 name: M-iBGP-101
[admin@M-iBGP-101] > ip address
[admin@M-iBGP-101] /ip address> add interface=ether1 address=13.1.2.101/24 disabled=no
[admin@M-iBGP-101] /ip address> add interface=ether2 address=13.1.5.101/24 disabled=no
[admin@M-iBGP-101] /ip address> add interface=ether3 address=13.1.8.101/24 disabled=no
[admin@M-iBGP-101] /ip address> add interface=ether4 address=13.1.9.101/24 disabled=no
[admin@M-iBGP-101] /ip address> add interface=ether5 address=13.1.6.101/24 disabled=no
[admin@M-iBGP-101] > interface bridge
[admin@M-iBGP-101] /interface bridge> add name=loopback0
[admin@M-iBGP-101] /ip address> add interface=loopback0 address=9.9.9.9/32 disabled=no

Настройка на C-iBGP-23:

Router>en
Router#conf t
Router(config)#hostname C-iBGP-23
C-iBGP-23(config)#int range e0/0-2
C-iBGP-23(config-if-range)#no shutdown
C-iBGP-23(config-if-range)#int e0/0
C-iBGP-23(config-if)#ip addr 13.1.4.23 255.255.255.0
C-iBGP-23(config-if)#int e0/1
C-iBGP-23(config-if)#ip addr 13.1.10.23 255.255.255.0
C-iBGP-23(config-if)#int e0/2
C-iBGP-23(config-if)#ip addr 13.1.8.23 255.255.255.0
C-iBGP-23(config-if)#do wr

Аналогично нужно сделать и со всеми остальными маршрутизаторами в зоне iBGP.

Настройка iBGP

Теперь перейдем к настройке протокола.

Настройка на C-iBGP-102:

C-iBGP-102(config)#router bgp 400
C-iBGP-102(config-router)#network 13.1.1.0 mask 255.255.255.0
C-iBGP-102(config-router)#network 13.1.2.0 mask 255.255.255.0
C-iBGP-102(config-router)#network 13.1.3.0 mask 255.255.255.0
C-iBGP-102(config-router)#neighbor 13.1.1.15 remote-as 400
C-iBGP-102(config-router)#neighbor 13.1.2.101 remote-as 400
C-iBGP-102(config-router)#neighbor 13.1.3.28 remote-as 400
  • network 13.1.1.0 mask 255.255.255.0 - указываем подключенные сети.
  • neighbor 13.1.1.15 remote-as 400 - создаем соседей.

Настройка на M-iBGP-101:

[admin@M-iBGP-101] > routing bgp instance
[admin@M-iBGP-101] /routing bgp instance > set default as=400 router-id=9.9.9.9
[admin@M-iBGP-101] > routing bgp network
[admin@M-iBGP-101] /routing bgp network> add network=13.1.2.0/24
[admin@M-iBGP-101] /routing bgp network> add network=13.1.5.0/24
[admin@M-iBGP-101] /routing bgp network> add network=13.1.8.0/24
[admin@M-iBGP-101] /routing bgp network> add network=13.1.9.0/24
[admin@M-iBGP-101] /routing bgp network> add network=13.1.6.0/24
[admin@M-iBGP-101] > routing bgp peer
[admin@M-iBGP-101] /routing bgp peer> add name=peer1 remote-address=13.1.2.102 remote-as=400 route-reflect=yes
[admin@M-iBGP-101] /routing bgp peer> add name=peer2 remote-address=13.1.6.28 remote-as=400 route-reflect=yes
[admin@M-iBGP-101] /routing bgp peer> add name=peer3 remote-address=13.1.5.15 remote-as=400 route-reflect=yes
[admin@M-iBGP-101] /routing bgp peer> add name=peer4 remote-address=13.1.8.23 remote-as=400 route-reflect=yes
[admin@M-iBGP-101] /routing bgp peer> add name=peer5 remote-address=13.1.9.33 remote-as=400 route-reflect=yes
Если настроить протокол BGP без route-reflect, то все маршрутизаторы не будут видеть сети дальше чем 2 hop, данная настройка позволяет решить эту проблему. В этом примере настройка route-reflect будет сделана только на MikroTik, это уже позволяет всем маршрутизаторам получать все маршруты, но при отключении M-iBGP-101 маршрутизаторы снова потеряют сети, располагающиеся за 2 hop. Вам стоит подумать над тем, как настроить сеть так, чтобы она была более устойчива.

Настройка на C-iBGP-23:

C-iBGP-23(config)#router bgp 400
C-iBGP-23(config-router))#network 13.1.4.0 mask 255.255.255.0
C-iBGP-23(config-router))#network 13.1.8.0 mask 255.255.255.0
C-iBGP-23(config-router))#network 13.1.10.0 mask 255.255.255.0
C-iBGP-23(config-router))#neighbor 13.1.4.15 remote-as 400
C-iBGP-23(config-router))#neighbor 13.1.8.101 remote-as 400
C-iBGP-23(config-router))#neighbor 13.1.10.33 remote-as 400

Посмотреть информацию о маршрутах можно при помощи следующих команд: * show ip route - просмотр всех маршрутов. * show ip bgp summary - просмотр соседей. * show ip route bgp - просмотр всех маршуров полученных по протоколу BGP.

Проверка правильности настройки для iBGP

  1. В зоне iBGP каждый роутер должен видеть все внутренние сети и пинговать все роутеры во внутренней сети.
  1. Не должно быть петель.
  1. Сеть должна быть устойчива, даже если отключено какое-либо устройство, сети должны быть видны за 2 и более hop.

Настройка eBGP

Настройка ip-адресации приводиться в этом разделе не будет. Перейдем сразу к настройке протокола BGP.

Настройка на M-OSPF-10:

[admin@M-OSPF-10] /routing bgp instance> set default as=300 redistribute-connected=yes redistribute-ospf=yes router-id=10.255.255.10
[admin@M-OSPF-10] /routing ospf instance> set default redistribute-bgp=as-type-1 redistribute-connected=as-type-1 router-id=10.255.255.10
[admin@M-OSPF-10] /ip address> add address=10.255.255.10 interface=loopback
[admin@M-OSPF-10] /ip address> add address=18.1.1.254/24 comment=C-RIP-1 interface=ether2
[admin@M-OSPF-10] /ip address> add address=14.1.1.254/24 comment=Router interface=ether1
[admin@M-OSPF-10] /ip address> add address=15.1.1.254/24 comment=C-iBGP-102 interface=ether6
[admin@M-OSPF-10] /routing bgp peer> add name=C-RIP-1 remote-address=18.1.1.1 remote-as=200
[admin@M-OSPF-10] /routing bgp peer> add name=Main-Router remote-address=14.1.1.1 remote-as=100
[admin@M-OSPF-10] /routing bgp peer> add name=C-iBGP-102 remote-address=15.1.1.1 remote-as=400
  • set default as=300 redistribute-connected=yes redistribute-ospf=yes router-id=10.255.255.10 - устанавливаем в default instanse, зона bgp - 300, что раздается информация о connected и ospf маршрутах, router-id - ip-адрес loopback.
  • set default redistribute-bgp=as-type-1 redistribute-connected=as-type-1 router-id=10.255.255.10 - устанавливаем в default instanse, раздается информация о connected и bgp маршрутах, router-id - ip-адрес loopback.
  • add name=C-RIP-1 remote-address=18.1.1.1 remote-as=200 - задется сосед в BGP, где remote-address - адрес соседа, name - имя соседа, remote-as - зона, в которой располагается сосед.

Настройка на C-RIP-1:

C-RIP-1(config)#router rip
C-RIP-1(config-router)#redistribute connected
C-RIP-1(config-router)#redistribute bgp 200
C-RIP-1(config-router)#default-metric 1
C-RIP-1(config-router)#exit
C-RIP-1(config)#router bgp 200
C-RIP-1(config-router)#redistribute connected
C-RIP-1(config-router)#redistribute rip
C-RIP-1(config-router)#neighbor 17.1.1.1 remote-as 100
C-RIP-1(config-router)#neighbor 18.1.1.254 remote-as 300
C-RIP-1(config-router)#default-metric 1
C-RIP-1(config-router)#exit
C-RIP-1(config)#exit
C-RIP-1#wr
  • redistribute connected - раздается информация о подключенных маршрутах.
  • redistribute bgp 200 - раздается информация о маршрутах BGP зоны 200.
  • redistribute rip - раздается информация о маршрутах RIP.
  • default-metric 1 - метрика для раздаваемых маршрутов.
  • neighbor 17.1.1.1 remote-as 10 - создаем соседа и указываем его зону.

Настройка на Main-Router:

Main-Router(config)#router bgp 100
Main-Router(config-router)#redistribute connected
Main-Router(config-router)#redistribute static
Main-Router(config-router)#neighbor 14.1.1.254 remote-as 300
Main-Router(config-router)#neighbor 16.1.1.254 remote-as 400
Main-Router(config-router)#neighbor 17.1.1.254 remote-as 200
Main-Router(config-router)#default-metric 1
Main-Router(config-router)#exit
Main-Router(config)#exit
Main-Router#wr
  • redistribute static - раздается информация о статических маршрутах.

Настройка на C-iBGP-102:

C-iBGP-102(config)#router bgp 400
C-iBGP-102(config-router)#network 16.1.1.0 mask 255.255.255.0
C-iBGP-102(config-router)#network 15.1.1.0 mask 255.255.255.0
C-iBGP-102(config-router)#redistribute connected
C-iBGP-102(config-router)#neighbor 13.1.2.1 next-hop-self
C-iBGP-102(config-router)# neighbor 15.1.1.254 remote-as 300
C-iBGP-102(config-router)#neighbor 16.1.1.1 remote-as 100
C-iBGP-102(config-router)#default-metric 1
C-iBGP-102(config-router)#exit
C-iBGP-102(config)#exit
C-iBGP-102#wr
  • neighbor 13.1.2.1 next-hop-self - эта команда позволяет решить проблему с микротиком. Суть проблемы: при просмотре на MikroTik таблицы маршрутизации можно обнаружить, что он знает о всех сетях, но все маршруты у него неактивны. Можете сами посмотреть и предложить сови варианты решений.

Проверка правильности настройки для eBGP

  1. Все устройства в лабораторной работе должны пинговать друг с друга.
  1. В области iBGP нужно посмотреть все роутеры на их знания о сетях. Убедиться в том, что все могут пинговать маршрутизаторы из других зон.
