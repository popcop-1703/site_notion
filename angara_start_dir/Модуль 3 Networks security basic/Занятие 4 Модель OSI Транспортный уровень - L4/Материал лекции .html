Понятие порта

Порт — абстракция, используемая протоколами транспортного уровня (TCP, UDP) для различения множества адресатов в пределах одного хоста. Список номеров портов

Существует 65536 портов TCP и UDP, разделенных на три основные группы:

  • Well-Known Ports — 0-1023
  • Registered Ports — 1024-49151
  • Dynamic & Private Ports — 49152-65535

Основные порты:

Флаги TCP

Для того чтобы управлять TCP соединением существуют флаги

Флаг Роль
SYN Синхронизация - первый флаг между двумя хостами, используется для установки соединения
ACK Подтверждение полученного пакета
RST Сброс - запрашивает разрыв соединения
FIN Сообщает другой стороне что все пакеты были отправлены
PSH Используется есть необходимо сразу отправить все данные из буфера
URG Указатель важности:
0 - не используется
1 - используется

Процесс установки соединения

Трехэтапное рукопожатие
  1. Клиент отправляет SYN
  1. Сервер отвечает SYN-ACK
  1. Клиект подтверждает соединение с помощью флага ACK

Процесс передачи данных

  1. Клиент отправляет серверу ACK-PSH, чтобы сервер не ждал продолжения полученных данных
  1. Сервер отвечает ACK
  1. Сервер обрабатывает запрос и отправляет клиенту данные (data)
  1. Клиент подтверждает что данные получены пакетом м флагом ACK

Процесс завершения сеанса

  1. Клиент запрашивает завершение сеанса, отправляя другой стороны флаг FIN
  1. Сервер возвращает флаг ACK для подтверждения запроса
  1. Сервер отправляет FIN
  1. Клиент утверждает FIN и отправляет ACK

Протокол UDP

Основные аспекты и отличия от TCP

  • Датаграммы
  • Неявные “рукопожатия”
  • Разработан в 1980 году
  • Ненадежность - дублирование, неупорядоченность, потери
  • Не подтверждает получение данных
  • При плохом соединении с возможностью потерь работает быстрее TCP

Принцип работы

  1. UDP получает сообщения от прикладного уровня
  1. Добавляет поля: номер порта отправителя и порта получателя, общую длину данных и контрольную сумму
  1. Передает семент уровню L3 (сетевой)
  1. Сетевой уровень заключает сегмент в датаграмму и передает хосту назначения
  1. Если сегмент успешно получен, протокол UDP направляет данные нужному процессу

TCP vs UDP

TCP UDP
Ориентирован на соединение Без установки соединения
Подходит для приложений, требующих высокой надежности, где время передачи менее критично Подходит для приложений которым требуется быстрая и эффективная передача
Упорядоченные пакеты данных в указаном порядке Пакеты могут приходить не по порядку, UPD их не упорядочивает
Размер заголовка составляет 20 байт Размер заголовка составляет 8 байт
Гарантирует передачу пакетов Нет гарантии передачи пакетов

Техники сканирования сети

Существует огромное количество утилит для сканирования сети. Обозначим некоторые из них:

  • Ping - эхо запрос, проверка достежимости хоста
  • Nmap - используется для обнаружения хостов и служб в компьютерной сети путем отправки пакетов и анализа ответов
  • Arpping - функционирует аналогично утилите ping, но, в отличие от неё, посылает/получает не ICMP-запросы/ответы, а ARP-запросы/ответы. Работает на уровне L2
  • Arp-scan - создает и отправляет ARP запросы указанным IP адресам и отображает полученные ответы
  • Masscan - массовый сканер IP портов
  • Nbtscan - инструмент для получения общих ресурсов NetBIOS и информации об именах
