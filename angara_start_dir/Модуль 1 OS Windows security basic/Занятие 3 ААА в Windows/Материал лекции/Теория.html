Права безопасности объектов в домене

Если заглянуть в свойства объекта домена, то там можно увидеть такие атрибуты как:

Основные атрибуты:

  • objectSID - идентификатор безопасности (SID) пользователя
  • AccountExpires - указывает на срок истечения действия учетной записи
  • PasswordExpirationDate - указывает на срок истечения действия пароля
  • BadPwdCount - сколько раз пользователь пытался войти в систему, использую неверный пароль
  • LastLogin - последний вход в систему
  • PwdLastSet - время последнего изменения пароля

Как происходит идентификация объекта в системе?

  1. Идентификация позволяет уникально определить объект безопасности (пользователь, группа, компьютер и тд), то есть проверяется есть ли такой объект в системе в принципе (проверяется objectSID)
  1. Аутентификация или проверка подлинности проверяет частную информацию объекта для аутентификации (проверка пароля или PIN)
  1. Авторизация подтверждает подлинность субъекта безопасности
  1. Отчетность позволяет сохранить события авторизации в предназначенные для этого журналы

Какой процесс в Windows позволяет ввести учетные данные?

Функции процесса:

  • Работоспособность и защита рабочего стола
  • Контроль хранения экрана (скринсейвер)
  • Загрузка пользовательский профилей
  • Обеспечения безопасности пользовательских данных
  • Поддержка входа и выхода пользователя из системы
После того как пользователь вводит пароль, он хешируется (за это ответственен процесс lsass.exe) и далее полученная строчка в локальной базе данных SAM сравнится с хешем, хранящимся там.

Netlogon

NTLMv2

Основные шаги:

  1. Из пароля пользователя в виде Unicode строки при помощи алгоритма MD4 вычисляется NTLM-hash, состоящий из 16 байт. Например, пароль «SecREt01» или Unicode строка 0x53006500630052004500740030003100 после преобразования посредством алгоритма MD4 будет иметь вид: 0xcd06ca7c7e10c99b1d33b7485a2ed808.
  1. Имя пользователя и имя домена в виде Unicode строк объединяются и приводятся к верхнему регистру. Далее, из этой последовательности и ранее полученного NTLM-hash при помощи алгоритма HMACMD5 вычисляется NTLMv2-hash, состоящий из 16 байт. Например, имя пользователя «User» и имя домена «Domain» дадут строку «USERDOMAIN» или Unicode строку 0x550053004500520044004f004d00410049004e00. После алгоритма HMACMD5, где применяется NTLM-hash, вычисляется NTLMv2-hash : 0x04b8e0ba74289cc540826bab1dee63ae.

Kerberos (протокол аутентификации в Windows)

Протокол, обеспечивающий надежную аутентификацию пользователей.

Данный протокол предусматривает, что обмен информацией между клиентом и сервером происходит в незащищенной среде, а передаваемые пакеты могут быть перехвачены. Поэтому, одна из самых важных задач, которые решает протокол - это подтверждение двух сторон через домен контроллер. Работает протокол с помощью билетов.

Кэширование данных в Windows

Чтобы посмотреть кеш, скачаем Sysinternals Suite - полезную коллекцию инструментов от Microsoft, которая поможет системным администраторам диагностировать и устранить проблему в различных областях - начиная от файловой системы, заканчивая сетью и настройками безопасности. И разархивируем программу под названием PsExec64.exe и запустим его через командную строку с флагами: PsExec64.exe -i -s regedit.exe.

В папке Cache мы сможем увидеть кешированные учетные данные пользователей.
