Отключение WDigest (не обязательно проделывать в рамках практики)

На современных системах уже не актуально, так как хранение пароля в открытом виде по умолчанию запрещается для WDigest начиная с Windows 8.1 и Server 2012 R2.
Если используются более старые системы, то необходимо установить специальное обновление — KB2871997, а после с помощью политики выставить параметры реестра для запрета хранения открытых паролей WDigest.

Защита LSA от подключения сторонних модулей

В Windows 8.1 и Windows Server 2012 R2 появилась возможность включить защиту LSA, которая защищает память LSA и предотвращает подключение к ней незащищенных процессов.
Чтобы включить эту защиту, вам необходимо создать параметр RunAsPPL со значением 1 в разделе реестра HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\LSA.

==4.18 Добавим в политику mimikatz_block_debug новый параметр реестра==

==4.19 Настроим параметр, активирущий защиту LSA==

ветка реестра: HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Control\LSA

имя параметра: RunAsPPL

Значение выставляем в 1. Внимание! Изменяем decimal значение

Теперь подключаемые модули без подписи, а также модули, не имеющие подписи Майкрософт, не будут загружаться в LSA
