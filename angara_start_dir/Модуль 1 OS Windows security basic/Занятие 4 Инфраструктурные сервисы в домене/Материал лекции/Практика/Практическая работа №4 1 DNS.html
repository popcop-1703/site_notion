Практическая работа №4.1 DNS.

Занятие 4 - Инфраструктурные сервисы в домене

Часть 1. DNS

Задача - настроить перенаправление запросов DNS, не относящихся к зоне pt.local, на внешний DNS сервер. В дополнение - сконфигурировать зону обратного просмотра.
Проверьте, что на микротике настроено перенаправление запросов - выполнены пункты 3.14 и 3.15 из инструкции
Занятие 0 - Подготовка стенда

1.1 Зайдём в оснастку DNS

1.2 Просмотрим текущие DNS записи

1.3 Настроим форвард. Перейдём в настройки сервера DNS

1.4 Откроем вкладку "forwarders"

1.5 Зайдём в пункт "edit"

1.6 Введём адрес mikrotik. Именно он будет перенаправлять DNS запросы на внешние сервера

1.7 Увидим, что настройка прошла корректно

Если вместо подтверждения увидите ошибку - проверьте, включен ли вам микротик и настроен ли на нём приём удалённых запросов, как указано вначале статьи

1.8 Закроем окно и применим настройки, после чего закроем окно свойств DNS

1.9 Настроим зону обратного просмотра. Нажмём ЛКМ на "reverse lookup zone", а после нажмём на неё ПКМ

1.10 Нажмём "new zone". Откроется меню создания зоны. NEXT

1.11 Оставляем пункт "primary zone". NEXT

1.12 Аналогично оставляем по умолчанию параметры репликации. NEXT

1.13 Настроим IPv4 зону. NEXT

1.14 Введём идентификатор зоны без последнего ip октета. Это наша сеть, для которой будет создана зона обратного просмотра. NEXT

1.15 Оставим пункт по умолчанию для динамического обновления зоны. NEXT

1.16 Сверимся с итоговыми настройками и завершим конфигурацию

1.17 Увидим созданную обратную зону
